# 篇幅估算参考指南

## 章节篇幅参考

| 章节类型 | 一级标题 | 二级标题 | 三级标题 |
|---------|---------|---------|---------|
| 项目概述 | 500-800字 | 300-500字 | 150-300字 |
| 现状分析 | 800-1200字 | 400-600字 | 200-400字 |
| 总体设计 | 1000-1500字 | 500-800字 | 300-500字 |
| 功能设计 | 1500-3000字/模块 | 500-1000字 | 300-500字 |
| 技术实现 | 1000-1500字 | 500-800字 | 300-500字 |
| 实施计划 | 600-1000字 | 400-600字 | 200-400字 |
| 保障措施 | 600-1000字 | 400-600字 | 200-400字 |

## 复杂度系数

| 复杂度 | 场景 | 系数 |
|-------|------|------|
| 低 | 单一功能、系统升级 | 0.7-0.8 |
| 中 | 多功能整合、新建系统 | 1.0 |
| 高 | 复杂业务、多系统集成 | 1.2-1.5 |
| 极高 | AI/大模型、跨行业创新 | 1.5-2.0 |

## 方案规模与篇幅对照

| 规模 | 功能模块数 | 参考篇幅 | 建议拆分 |
|-----|-----------|---------|---------|
| 小型 | 1-2个 | 5000-8000字 | 不拆分或按二级拆分 |
| 中小型 | 3-4个 | 8000-12000字 | 按一级拆分4-5个子任务 |
| 中型 | 5-7个 | 12000-20000字 | 按一级拆分7-10个子任务 |
| 大型 | 8-12个 | 20000-30000字 | 分批执行，每批5-7个 |
| 超大型 | 12+个 | 30000-50000字 | 分批执行，每批5-7个 |

## 任务拆分检查清单

- [ ] 单个子任务字数 < 6000字
- [ ] 单个子任务字数 > 3000字（避免过细）
- [ ] 一级标题超过10000字 → 拆分为二级
- [ ] 二级标题超过6000字 → 拆分为三级
- [ ] 上下文可独立理解
- [ ] 任务间无循环依赖
