# 质量检查清单

## 使用说明

本文档用于技术标书写作质量的系统化检查。按章节逐项检查，确保文档符合规范要求。

---

## 1. 写作风格检查

### 1.1 人称使用

| 检查项 | 状态 | 备注 |
|-------|------|-----|
| 是否避免使用"我们" | ☐ | |
| 是否避免使用"你/你们" | ☐ | |
| 是否使用无主句或以方案为主语 | ☐ | |

**常见问题示例**：
- ❌ "我们选择微服务架构" → ✅ "微服务架构被选定"
- ❌ "你需要配置" → ✅ "需配置"

### 1.2 标题规范

| 检查项 | 状态 | 备注 |
|-------|------|-----|
| 标题是否使用陈述句 | ☐ | 无问句标题 |
| 标题编号是否连续 | ☐ | X.Y.Z格式 |
| 标题层级是否合理 | ☐ | 无跳跃 |
| 五级标题是否必要 | ☐ | 尽量避免 |

**禁止的标题形式**：
- ❌ "为什么需要X？"
- ❌ "X解决什么问题？"
- ❌ "如何实现X？"

### 1.3 论证逻辑

| 检查项 | 状态 | 备注 |
|-------|------|-----|
| 是否避免"放弃的方案"表述 | ☐ | |
| 是否采用对比分析模式 | ☐ | |
| 是否包含What-Why-How三要素 | ☐ | |
| 是否避免绝对化表述 | ☐ | |

### 2.3 客观性

| 是否避免夸大表述 | ☐ | 无"国际领先"等 |

| 检查项 | 状态 | 问题位置 |

| 三级标题格式正确 | ☐ | ### X.Y.Z |
| 四级标题格式正确 | ☐ | #### X.Y.Z.W |

### 4.2 表格规范

| 检查项 | 状态 | 问题位置 |
|-------|------|---------|
| 表格有编号 | ☐ | 表X-Y |
| 表格有标题 | ☐ | |
| 表格有注释（如需要） | ☐ | |
| 单位已标明 | ☐ | |

### 4.3 图表规范

| 检查项 | 状态 | 问题位置 |
|-------|------|---------|
| 图表有编号 | ☐ | 图X-Y |
| 图表有标题 | ☐ | |
| 占位符已处理 | ☐ | 无【】占位 |
| 图表被正文引用 | ☐ | |

### 4.4 无序列表规范（重要）

| 检查项 | 状态 | 问题位置 |
|-------|------|---------|
| 无序列表数量是否合理 | ☐ | 全文≤全文行数5% |
| 单个列表条目数是否适当 | ☐ | ≤5条 |
| 是否避免用列表替代段落论述 | ☐ | |
| 技术选型是否用段落而非列表 | ☐ | |

**常见问题**：
- ❌ 技术栈用列表罗列 → ✅ 整合为连贯段落
- ❌ 功能模块用列表描述 → ✅ 改为描述性段落
- ❌ 配置参数用列表列举 → ✅ 整合为描述性文字

### 4.5 "以表代文"检查（重要）

| 检查项 | 状态 | 问题位置 |
|-------|------|---------|
| 表格前是否有充分文字说明 | ☐ | ≥3句背景分析 |
| 表格后是否有结论阐述 | ☐ | ≥2句理由说明 |
| 技术选型是否有完整论证 | ☐ | 非仅引入语 |
| 方案对比是否以文字为主 | ☐ | 表格仅辅助 |

**检查方法**：
1. 定位所有对比类/选型类表格
2. 检查表格前文字：是否有充分的背景分析、问题引入
3. 检查表格后文字：是否有充分的结论阐述、选择理由
4. 评估：主要论证内容是否在表格中（应改为文字论述）

### 4.6 其他格式

| 检查项 | 状态 | 问题位置 |
|-------|------|---------|
| 分隔线使用合理 | ☐ | 不过度使用 |
| 列表层级≤3级 | ☐ | |
| 数字单位格式正确 | ☐ | 100 ms |

## 7. 快速检查口诀

**写作三避免**：
- 避免第一人称
- 避免问句标题
- 避免负面表述

**语言三要求**：
- 正式不口语
- 准确不模糊
- 客观不夸大

**证据三要素**：
- 有数值范围
- 有对比基准
- 有数据来源

**格式五规范**：
- 标题层级清
- 图表编号明
- 分隔不滥用
- **列表不过度**（≤全文5%）
- **表格不代文**（文字论述为主）
