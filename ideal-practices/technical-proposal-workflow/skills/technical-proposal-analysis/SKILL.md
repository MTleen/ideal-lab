# Technical Proposal Analysis (P1)

## Overview

解析需求文档，提取功能需求、非功能需求、技术约束，生成结构化的需求分析文档。

## 输入

- 需求文档（用户提供）
- 项目目录和方案名称

## 输出

`{项目目录}/项目文档/技术方案/P1-需求分析.md`

## 执行步骤

### 1. 读取需求文档

解析用户提供的需求文档，识别以下字段：

**功能需求：**
- 场景/功能名称
- 目标（Goal）
- 当前状态（Current State）
- 输入（Input）
- 输出（Output）
- 价值/效果（Value）

**非功能需求：**
- 性能要求
- 安全要求
- 可靠性要求
- 兼容性要求

**技术约束：**
- 技术栈限制
- 集成要求
- 合规要求

### 2. 生成需求分析文档

```markdown
# P1 需求分析

## 项目信息
- 项目名称：{方案名称}
- 分析时间：{时间}

## 功能需求

### 场景：{场景名称}
- **目标**：{目标描述}
- **当前状态**：{现状描述}
- **输入**：{输入描述}
- **输出**：{输出描述}
- **价值**：{价值描述}

## 非功能需求
- 性能要求：{描述}
- 安全要求：{描述}
- 可靠性要求：{描述}

## 技术约束
- 技术栈：{描述}
- 集成要求：{描述}
- 合规要求：{描述}

## 复杂度评估
- 功能模块数：X
- 预估总篇幅：XXXX字
- 建议拆分：X个子任务
```

### 3. 保存文件

保存到 `{项目目录}/项目文档/技术方案/P1-需求分析.md`

## 示例

### 输入

```
场景：IT运维助手
目标：1. 对运维情况进行评估，了解当前状态；2. 对历史运维工单进行分析生成运维隐患报告及优化建议；3. 给常见问题知识问答库编写建议。
当前状态：人工整理运维工单...
输入：读取报警数据、工单数据、电话录音
输出：运维日报、隐患报告及知识库编写建议
```
