# P1 需求分析（科普版）

## 文档信息
- 文档名称：云端技术怎样助力人工智能高效落地
- 文档类型：知识普及文章
- 目标读者：大众（无技术背景的普通读者）
- 输出格式：docx
- 预计阅读时长：约30分钟
- 分析时间：2026-05-12

## 需求来源

基于上一版企业课程讲义的调整需求：受众从企业员工改为普通大众，篇幅缩减至5000-7000字，风格改为纯科普。

---

## 功能需求（内容模块）

### 模块1：引子——AI就在你身边
- **目标**：用一个日常场景切入，让读者立刻感受到AI与自己的生活紧密相关
- **输入**：日常生活中的AI应用场景（手机拍照、语音助手、个性化推荐、智能客服）
- **输出**：读者意识到"原来AI已经无处不在"，产生了解背后的好奇心
- **价值**：打破"AI是高科技离我很远"的心理距离

### 模块2：AI为什么需要"云"
- **目标**：用通俗的类比讲清AI训练和推理对算力的巨大需求
- **输入**：大模型算力需求（训练=培养博士生、推理=博士生回答问题）、自建算力vs云端算力
- **输出**：读者理解"AI很吃算力，只有云能经济地满足这种需求"
- **价值**：建立核心认知——"没有云，就没有今天的AI"

### 模块3：云端到底做了什么
- **目标**：用3-4个生活类比讲清云端为AI提供的核心能力
- **输入**：算力调度（快递分拣）、资源协同（后厨团队）、数据管理（食品质检）、部署上线（新品上架）
- **输出**：读者能用自己的话描述云端为AI做了哪些事
- **价值**：从"云是模糊概念"升级为"云是一套精密的支撑系统"

### 模块4：云+AI融合的全景
- **目标**：用一个贯穿的类比呈现云智融合的完整画面
- **输入**：四层架构（用建筑类比）、MaaS（选菜谱/改菜谱/用菜谱）、AI智能体（实习生）
- **输出**：读者对"云智融合"有完整的画面感
- **价值**：建立系统性认知，不再停留在"云就是远程服务器"

### 模块5：这跟你有什么关系
- **目标**：用真实案例展示云智融合如何改变普通人的生活和工作
- **输入**：电信/金融/制造案例（来自调研报告）、普通人能感知的变化
- **输出**：读者感受到云智融合正在影响自己的生活，并且未来会更多
- **价值**：从"知道了"到"跟我有关了"

## 非功能需求

| 需求类型 | 描述 | 写作检查项 |
|----------|------|------------|
| **零门槛** | 任何背景的读者都能顺畅阅读，不需要任何技术基础 | 全文不出现未解释的技术术语；每个术语首次出现时用日常类比解释 |
| **故事驱动** | 用场景和故事引入概念，而非先给定义再举例 | 每个模块以一个场景/故事开头 |
| **类比密度** | 每个核心概念至少有一个生活化类比 | 类比来自日常生活（烹饪、建筑、出行、购物等） |
| **篇幅** | 5000-7000字，约30分钟阅读 | 不超过7000字 |
| **节奏** | 每个模块800-1500字，读完一个小节能休息 | 模块长度均匀 |
| **无数据轰炸** | 数据点控制在每个模块不超过3个，且数据服务于叙事 | 避免密集列举数字 |

## 技术约束

| 约束类型 | 描述 |
|----------|------|
| **语言** | 中文，口语化但不随意 |
| **风格** | 像一个聪明的朋友在跟你聊天，不是教科书在教你 |
| **术语处理** | 零容忍——出现任何技术术语都必须立即用类比或通俗语言解释 |
| **配图** | 3-5张示意图（架构图、流程图），配图用文字描述清楚 |
| **范围排除** | 不涉及具体产品对比、不涉及编程/部署实操、不涉及定价分析 |

## 复杂度评估
- **功能模块数**：5个
- **预估总篇幅**：5000-7000字
- **建议拆分**：2个写作任务（认知篇：模块1+2+3，实践篇：模块4+5+结语）
- **难度评估**：中等偏高——大众科普对类比精准度和叙事流畅度要求极高
- **信息充分度**：高——已有调研报告+上一版讲义可参考
- **预估配图数量**：3-5张
