# 量化证据要求

## 1. 总原则

评审不反感"好结果"，反感的是：
- **没有对照**：只说"更好/领先"，但不说明比谁好、好多少
- **没有证据**：没有数据、没有引用、没有实验设计支撑
- **自我定性**：用形容词替代论证（重大/关键/突破），且缺少可验证锚点

---

## 2. 数值表述规范

### 2.1 必须包含的要素

每个量化指标应包含以下要素：

| 要素 | 说明 | 示例 |
|-----|------|-----|
| 具体数值 | 明确的数字 | 88% |
| 范围/置信区间 | 体现不确定性 | 88%-92% |
| 数据来源 | 数据来自何处 | 基于测试集验证 |
| 样本量 | 数据规模 | n=1200 |
| 适用条件 | 在什么条件下成立 | 在X场景下 |

### 2.2 数值表述模板

**模板1：带范围的表述**
```
[指标]达到[下限]-[上限]（[数据来源]，n=[样本量]）
```
示例：准确率达到88%-92%（基于历史数据回测，n=1200）

**模板2：带对比的表述**
```
较[基准]提升/降低[数值]（[数据来源]）
```
示例：较传统数值模式（基准准确率65%）提升15个百分点

**模板3：带条件的表述**
```
在[条件]下，[指标]达到[数值]
```
示例：在日均调用量10万次场景下，响应延迟稳定在200ms以内

## 3. 对比证据规范

### 3.1 对比的三个维度

每次对比应明确以下三个维度：

| 维度 | 说明 | 示例 |
|-----|------|-----|
| 对比对象 | 与什么进行对比 | 与传统数值模式对比 |
| 对比指标 | 对比的具体指标 | 准确率、响应时间 |
| 对比结果 | 量化的差异 | 提升15个百分点 |

### 3.2 对比表述模板

**基准对比**
```markdown
相比[基准方案]，[指标]提升[X]%（[验证方式]）
```

**多方案对比**
```markdown
| 方案 | 指标A | 指标B | 说明 |
|-----|------|------|-----|
| 方案1 | X% | Yms | [基准] |
| 方案2 | (X+Z)% | (Y-W)ms | 提升[数据来源] |
| 本方案 | (X+Z+M)% | (Y-W-N)ms | [验证方式] |
```

### 3.3 对比禁忌

| ❌ 禁止 | ✅ 正确做法 |
|--------|-----------|
| "相比传统方案大幅提升" | "相比传统方案（准确率65%）提升15个百分点" |
| "性能更优" | "响应时间从500ms降低至200ms" |
| "效率更高" | "处理效率从100条/秒提升至500条/秒" |

## 4. 数据来源规范

### 4.1 可接受的数据来源

| 来源类型 | 说明 | 可信度 |
|---------|------|--------|
| 实测数据 | 本项目实际测试结果 | 高 |
| 历史数据 | 基于历史数据的统计分析 | 高 |
| 行业报告 | 权威机构发布的行业数据 | 中 |
| 学术论文 | 经同行评审的研究结果 | 中 |
| 厂商数据 | 技术厂商官方数据 | 中（需注明来源） |
| 估算数据 | 基于合理假设的估算 | 低（需明确说明为估算） |

### 4.2 数据来源标注

**必须标注来源的情况**：
- 引用外部数据
- 使用行业基准
- 采用第三方测试结果

**标注格式**：
```
（数据来源：[来源名称]，[时间]）
（基于[机构]发布的《[报告名称]》）
```

### 4.3 估算数据规范

当使用估算数据时，必须：
1. 明确说明为"预估"或"预计"
2. 说明估算依据和假设条件
3. 给出合理的范围而非精确值

**示例**：
```markdown
预计日均调用量约10万次（基于当前500用户规模及历史增长率估算，
实际可能波动±20%）
```

## 5. 可核验性要求

### 5.1 可核验的表述

| ❌ 不可核验 | ✅ 可核验 |
|-----------|----------|
| 效果显著 | 准确率提升10个百分点（p<0.05） |
| 性能优越 | 响应时间<200ms（95分位） |
| 大幅提升 | 处理效率从X提升至Y |
| 国际领先 | 在Z数据集上达到SOTA水平 |

### 5.2 可核验的要素

**统计指标**：
- 均值、中位数、标准差
- 置信区间（95% CI）
- 显著性水平（p值）
- 效应量（Cohen's d等）

**测试条件**：
- 数据集规模
- 测试环境
- 评估方法
- 对比基准

### 5.3 可核验表述模板

```markdown
在[测试环境]下，基于[数据集]（n=[样本量]）的测试结果表明：
- [指标1]：[均值]±[标准差]，95% CI: [下限,上限]
- 相比[基准]，提升[数值]（p<[显著性水平]）
```

## 6. 常见问题与改写

### 6.1 无具体数值

| ❌ 问题表述 | ✅ 改写 |
|-----------|--------|
| 准确率较高 | 准确率达到92% |
| 响应很快 | 响应时间<100ms |
| 大量数据 | 日均处理数据量超过100万条 |
| 显著提升 | 提升约30% |

### 6.2 无对比基准

| ❌ 问题表述 | ✅ 改写 |
|-----------|--------|
| 达到90%准确率 | 较基线模型（准确率75%）提升15个百分点 |
| 处理速度快 | 较传统方案（500ms）提升60%，降至200ms |
| 成本低 | 较方案A降低成本约40% |

### 6.3 无来源说明

| ❌ 问题表述 | ✅ 改写 |
|-----------|--------|
| 预计可达90% | 预计可达90%（基于小规模试验结果外推） |
| 调用量大 | 日均调用量约10万次（基于现有用户规模估算） |
