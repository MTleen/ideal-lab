# Task Split (P3)

## Overview

根据大纲将文档拆分为独立的写作任务，每个任务约5000字，可并行执行。

## 输入

- P2-大纲.md
- 项目目录和文档名称

## 输出

`{项目目录}/文档产出/P3-任务拆分.md`

## 拆分规则

1. **优先按一级标题拆分**：每个一级标题 = 一个子任务
2. **触发条件**：若单个一级标题预估 >10000字 → 按二级标题拆分
3. **递归检查**：若二级标题仍 >10000字 → 按三级标题拆分
4. **最小单元**：每个子任务控制在 3000-6000字（目标5000字）

## 任务卡片格式

```markdown
### 任务 1: 1. 概述
- **文件**：`sections/1-概述.md`
- **预估字数**：800字
- **章节层级**：1.1 ~ 1.3
- **上下文**：
  - 来源需求：{需求片段}
  - 上级概述：{大纲中的一句话概述}
- **写作要点**：
  - 背景需要交代行业背景和客户痛点
  - 目标要可量化、可衡量

### 任务 2: 2. 现状分析
- **文件**：`sections/2-现状分析.md`
- **预估字数**：1200字
- **章节层级**：2.1 ~ 2.4
- **上下文**：...
- **写作要点**：...
```

## 上下文打包

每个子任务的上下文包含：
1. 原始需求文档中相关内容
2. 上一级大纲中的描述
3. 同级其他任务的标题（避免重复）
4. 特殊写作要求
