# TDD 协议

## 铁律

**测试先行**：必须先写失败的测试，再写实现代码。

## TDD 循环

```
RED:   编写失败的测试 → 运行确认失败
GREEN: 编写最小实现 → 运行确认通过
REFACTOR: 清理代码 → 保持测试通过
COMMIT: 提交代码
```

## 每次任务执行

1. **RED 阶段**：根据任务验收标准编写测试，确认测试失败
2. **GREEN 阶段**：编写最小实现代码使测试通过
3. **REFACTOR 阶段**：清理代码，保持测试通过
4. **COMMIT**：确认测试全部通过后提交

## 注意事项

- 一次只关注一个测试用例
- 实现代码只要求通过测试，不求完美
- 重构只在 GREEN 阶段之后进行
- 提交信息需说明验证了哪个验收标准
