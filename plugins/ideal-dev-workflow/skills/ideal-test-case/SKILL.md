# ideal-test-case（P7 测试用例生成）

## 角色定位

Phase Skill — **执行协调者**。

职责：
1. 加载所需上下文（P1 需求 + P5 编码计划）
2. 调度子智能体生成测试用例
3. 将产物写入文件系统
4. 返回执行摘要

**不负责**：更新 flow state、验证前置条件、协调评审。

---

## 输入

| 来源 | 内容 |
|------|------|
| P1-需求文档.md | 验收标准、功能需求 |
| P5-编码计划.md | 任务清单、功能点映射 |
| stories/index.md | 故事列表、executor 标签（用于按 story 组织测试用例） |

---

## 输出

| 文件 | 路径 |
|------|------|
| P7-测试用例.md | `docs/迭代/{需求名称}/P7-测试用例.md` |

---

## 子智能体调度

| 调用时机 | 子智能体 | 任务 |
|----------|----------|------|
| 验收标准提取 | `qa` | 从 P1 中提取验收标准作为用例基础 |
| 功能用例生成 | `qa` | 覆盖主流程和分支流程 |
| 边界用例生成 | `qa` | 输入边界、状态边界、流程边界 |
| 异常用例生成 | `qa` | 错误输入、异常状态、外部错误 |

## 测试用例格式

### 功能/边界/异常测试

```markdown
## TC-{type}-{id}: {用例名称}

| 项目 | 内容 |
|------|------|
| **用例编号** | TC-{type}-{id} |
| **用例名称** | {名称} |
| **关联故事** | Story {id}（从 stories/index.md 映射） |
| **优先级** | P0 / P1 / P2 |
| **前置条件** | {测试前需满足的条件} |
| **来源** | P1-需求文档.md #{章节} / P5-编码计划.md T{xxx} |

**测试步骤**:
1. {步骤1}
2. {步骤2}

**预期结果**:
{期望的输出或状态}
```

**类型前缀**：`F` = 功能测试，`B` = 边界测试，`E` = 异常测试

### 前端截图测试（必须）

> **强制要求**：任何涉及前端 UI 开发的迭代，P11 测试执行时必须通过 Chrome DevTools MCP 对主要功能进行截图验证，不得仅依赖 TypeScript 编译检查或手工截图。

**触发条件**：P5 编码计划中存在前端任务（如 React/Vue 组件、页面路由、Store 状态等）。

**截图方式**：通过 Chrome DevTools MCP 工具驱动浏览器，Agent 直接控制导航、交互和截图。无需编写任何测试脚本，截图分辨率自动匹配当前屏幕。

**截图保存规范**：

| 项目 | 规范 |
|------|------|
| 截图保存目录 | `docs/迭代/{需求名称}/screenshots/` |
| 截图命名规则 | `TC-S-{id}_{名称}.png` |
| 视口大小 | 自动使用当前屏幕分辨率（无需配置） |
| 截图方式 | 全页面截图或指定元素截图（由 Agent 通过 MCP 工具操作） |
| 工具 | Chrome DevTools MCP（Agent 直接调用，零额外代码） |

**前端截图用例格式**：

```markdown
### TC-S-{id}: {截图测试名称}

| 项目 | 值 |
|------|-----|
| 用例编号 | TC-S-{id} |
| 优先级 | P0 / P1 / P2 |
| 测试步骤 | {描述 Agent 应执行的操作} |
| 预期结果 | 截图保存到 `docs/迭代/{需求名称}/screenshots/TC-S-{id}_{name}.png` |
| 截图结果 | ✅ 已保存 / ❌ 未保存 |
| 备注 | {说明} |
```

**前端截图测试覆盖要求**：

| 组件类型 | 必须截图项 |
|----------|------------|
| 表单组件 | 空状态、填写中、验证错误、提交成功 |
| 数据展示组件 | 空状态、加载中、数据正常、异常状态 |
| 交互组件 | 初始状态、交互后状态、动画过程中 |
| 页面 | 完整视口截图（自动使用当前屏幕分辨率）|
| API 组件 | 调用前、调用中（流式）、调用完成 |

## 执行流程

```
Step 0: 验证前置产物
  ├─ 检查 P1-需求文档.md 存在 → 否则终止
  ├─ 检查 P5-编码计划.md 存在 → 否则终止
  └─ 检查 stories/index.md 存在 → 否则终止

Step 1: 加载上下文
  ├─ 读取 P1-需求文档.md（提取验收标准、功能需求）
  ├─ 读取 P5-编码计划.md（提取任务和功能点）
  └─ 读取 stories/index.md（提取故事列表、executor 标签）

Step 2: 验收标准提取
  └─ Task(qa) → 将验收标准映射为可测试用例

Step 3: 功能用例生成
  └─ Task(qa) → 每个功能点至少一个正向用例，覆盖主流程

Step 4: 边界用例生成
  └─ Task(qa) → 输入/状态/流程边界用例

Step 5: 异常用例生成
  └─ Task(qa) → 错误输入、异常状态、外部错误用例

Step 6: 前端截图用例生成（仅当 P5 包含前端任务时）
  ├─ 确定前端组件/页面列表
  ├─ 创建截图保存目录 `docs/迭代/{需求名称}/screenshots/`
  ├─ 生成 CDP 截图测试用例
  └─ 确定截图覆盖项（空状态、交互后、完整页面）

Step 7: 覆盖率统计
  └─ 统计各类型用例数量，计算覆盖率

Step 8: 写入产物
  └─ 写入 P7-测试用例.md

Step 9: 返回摘要
  └─ 报告：用例数量、覆盖率、优先级分布
```

---

## 覆盖率目标

| 类型 | 占比 | 说明 |
|------|------|------|
| 功能测试 (F) | 50-60% | 验证正常功能流程 |
| 边界测试 (B) | 20-30% | 验证输入/状态边界 |
| 异常测试 (E) | 15-20% | 验证错误处理 |

---

## 质量检查清单

产物写入前必须验证：

- [ ] 每个验收标准至少有一个对应用例
- [ ] 包含功能、边界、异常三类用例
- [ ] 用例可独立执行（无执行顺序依赖）
- [ ] 预期结果明确（可判断 PASS/FAIL）
- [ ] 前置条件清晰
- [ ] 用例编号连续无重复
- [ ] P0 用例全部为正向功能用例
- [ ] **若 P5 包含前端任务：必须包含截图测试用例**
- [ ] **截图测试覆盖：空状态、完整页面、关键交互**
