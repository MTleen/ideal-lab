# 旁白写作指南

## 核心原则

旁白不是朗读幻灯片。旁白是老师在讲解幻灯片——学生能看见画面，你需要补充的是画面上没有的东西。

## 语气和风格

### 用口语，不用书面语

| ❌ 书面语 | ✅ 口语 |
|-----------|---------|
| 因此 | 所以 |
| 该方法 | 这个方法 |
| 接下来将介绍 | 我们来看看 |
| 上述 | 刚才说的 |
| 需要注意的是 | 注意一点 |
| 综上所述 | 总结一下 |

### 基本规则

- 第二人称："你可以看到..."、"你只需要..."
- 主动语态，现在时
- 短句为主（每句 15-25 字）
- 自然过渡，避免"第一...第二...第三..."的机械罗列

## 长度指南（按幻灯片类型）

| 类型 | 字数范围 | 示例 |
|------|----------|------|
| 封面 | 30-50 字 | "大家好，欢迎来到{课程名}。今天我们一起来学习{主题}。" |
| 分节页 | 40-80 字 | 简要预告接下来的内容 |
| 内容页 | 100-200 字 | 解释要点，补充幻灯片没有的上下文 |
| 代码页 | 60-120 字 | 解释代码做了什么、为什么重要 |
| 总结页 | 80-120 字 | 回顾关键收获 |

**每页不超过 250 字**——超过会导致单页停留太久，节奏拖沓。

## 过渡句（从第 2 页开始使用）

- "接下来我们看看..."
- "现在我们来了解..."
- "下面是一个重要的概念..."
- "我们进入下一节..."
- "讲完了 X，我们来看看 Y..."
- "最后我们总结一下..."
- "到这里，我们再来回顾..."

## 不要做的事

1. **不要逐字念幻灯片**——观众已经看到了，你念一遍是浪费时间
2. **不要说"这一页展示了..."或"如图所示..."**——画面就是证据，不需要声明
3. **不要使用未解释的术语**——如果是新概念，用半句话交代背景
4. **不要重复标题**——标题已经在了，直接进入解释

## 正反示例

### 示例 1：安装步骤

幻灯片内容：
- 下载 Node.js 安装包
- 运行安装程序
- 验证：`node --version`

❌ 差的旁白：
> "这一页展示了安装 Node.js 的步骤。第一步是下载安装包。第二步是运行安装程序。第三步是验证安装，输入 node --version。"

✅ 好的旁白：
> "首先我们需要安装 Node.js。去官网下载对应系统的安装包，一路下一步就行。装好之后打开终端输入 node --version，看到版本号就说明安装成功了。"

### 示例 2：概念对比

幻灯片内容：
- REST vs GraphQL 对比表

❌ 差的旁白：
> "这一页展示了 REST 和 GraphQL 的对比。REST 的特点是...GraphQL 的特点是..."

✅ 好的旁白：
> "REST 和 GraphQL 怎么选？简单说，如果你的数据关系比较简单，REST 就够了。但如果你的页面需要一次请求拿多种关联数据，GraphQL 会方便很多。不用纠结，先用你熟悉的，需要的时候再切换。"

## 实用技巧

1. **先写要点，再润色**——先把每页要传达的 1-2 个核心信息写下来，然后扩写成口语
2. **读出来测试**——在心里默读一遍，觉得别扭的地方就改
3. **保持节奏**——内容页之间穿插简短的过渡，避免连续多页密集信息
4. **加入个人感**——"我个人推荐..."、"最常用的方法是..."，比纯客观描述更有吸引力
