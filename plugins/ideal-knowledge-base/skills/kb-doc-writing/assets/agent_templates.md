## 决策流程

### 核心决策循环

```mermaid
flowchart TD
    INPUT["接收用户输入"] --> UNDERSTAND["理解意图<br/>（Intent Classification）"]
    UNDERSTAND --> PLAN{"是否需要规划？"}
    PLAN -->|简单任务| EXEC["直接执行"]
    PLAN -->|复杂任务| DECOMPOSE["任务分解<br/>（Decomposition）"]
    DECOMPOSE --> SUBTASKS["生成子任务列表"]
    SUBTASKS --> EXEC

    EXEC --> ACTION{"选择行动"}
    ACTION -->|调用工具| TOOL["工具调用"]
    ACTION -->|生成回复| REPLY["LLM 生成回复"]
    ACTION -->|追问用户| CLARIFY["向用户澄清"]

    TOOL --> OBSERVE["观察工具返回结果"]
    OBSERVE --> REFLECT{"结果是否满意？"}
    REFLECT -->|是| REPLY
    REFLECT -->|否| RETRY["调整策略重试"]
    RETRY --> ACTION

    CLARIFY --> INPUT
    REPLY --> EVAL{"需要自我评估？"}
    EVAL -->|是| SELF_CHECK["自我校验"]
    EVAL -->|否| OUTPUT["输出最终回复"]
    SELF_CHECK -->|"通过"| OUTPUT
    SELF_CHECK -->|"不通过"| REVISE["修正回复"]
    REVISE --> OUTPUT
```

| 边界类型 | 边界说明 | 超出边界的处理 |
|----------|----------|---------------|
