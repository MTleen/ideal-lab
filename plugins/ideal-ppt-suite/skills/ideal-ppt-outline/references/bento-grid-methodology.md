# Bento Grid 布局方法论

## 核心思想

Bento Grid（便当盒网格）是一种卡片式布局系统。每页幻灯片被视为一个网格容器，内容被组织为大小不一的矩形卡片。这是 AI 最容易理解和生成的布局方式。

**为什么有效**：
1. AI SVG 生成器能一致地渲染矩形卡片，而复杂有机布局经常出错
2. 卡片大小自然传递信息层级（大卡片 = 重要信息）
3. 组合灵活，能适应几乎所有内容类型
4. 视觉一致性强，跨页风格统一

## 卡片类型

| 卡片类型 | 内容 | 典型尺寸 | 视觉特征 |
|----------|------|---------|----------|
| **标题卡片** | 主要信息、核心论点 | 大（占 1/2 到全宽） | 醒目背景色，大字体 |
| **内容卡片** | 2-4 个要点 | 中等（1/2 到 1/3 宽） | 白色/浅色背景，列表项 |
| **数据卡片** | 数字/指标 + 标签 | 小到中等 | 大号数字，对比色标签 |
| **图片卡片** | 插图/图表占位 | 任意 | 图片填充或虚线占位框 |
| **引用卡片** | 特色文本 + 出处 | 中等 | 特殊排版（大引号/斜体） |

## 布局模式

### 1. 单一焦点
```
┌─────────────────────────────┐
│                             │
│       一张大卡片              │
│     (w=1200, h=580)         │
│                             │
└─────────────────────────────┘
```
适用于：单一有力信息、详细图表、封面

### 2. 两栏对称
```
┌──────────────┬──────────────┐
│              │              │
│   卡片 A     │   卡片 B     │
│  (580x560)   │  (580x560)   │
│              │              │
└──────────────┴──────────────┘
```
适用于：对比分析、并列论点

### 3. 两栏非对称
```
┌────────────────────┬───────────┐
│                    │           │
│    主卡片          │  辅助卡片  │
│   (780x560)        │ (380x560) │
│                    │           │
└────────────────────┴───────────┘
```
适用于：主内容 + 辅助信息/数据

### 4. 三栏
```
┌──────────┬──────────┬──────────┐
│          │          │          │
│  卡片 A  │  卡片 B  │  卡片 C  │
│ (380x560)│ (380x560)│ (380x560)│
│          │          │          │
└──────────┴──────────┴──────────┘
```
适用于：并列比较、三步流程

### 5. Hero + 网格
```
┌─────────────────────────────┐
│     英雄卡片 (w=1200, h=280) │
├──────────┬──────────┬────────┤
│ 卡片 B   │ 卡片 C   │ 卡片 D │
│(380x260) │(380x260) │(380x260)│
└──────────┴──────────┴────────┘
```
适用于：顶部关键信息 + 下方支撑数据

### 6. 主次结合
```
┌──────┬────────────────┬──────┐
│      │                │      │
│ 小卡 │    大卡片       │ 小卡 │
│(280) │    (600x560)    │(280) │
│      │                │      │
└──────┴────────────────┴──────┘
```
适用于：核心居中 + 两侧辅助

### 7. 混合网格
```
┌──────────────┬──────────────┐
│              │   卡片 B     │
│  卡片 A      │  (380x270)   │
│  (580x270)   ├──────────────┤
│              │   卡片 C     │
├──────────────┤  (380x270)   │
│  卡片 D      │              │
│  (580x270)   ├──────────────┤
│              │   卡片 E     │
└──────────────┘  (380x270)   │
                 └──────────────┘
```
适用于：多样化内容，自由度最高

## 设计规则

| 规则 | 值 |
|------|-----|
| 卡片间距 | ≥ 20px（推荐 24px） |
| 圆角 | 8-16px（全文档统一） |
| 卡片背景 | 白色或主色的浅色调 |
| 卡片阴影 | 微妙（可选，用偏移 rect 实现） |
| 安全边距 | 40px（画布边缘到卡片边缘） |
| 每页卡片数 | 1-7 个（取决于信息密度） |

## 卡片内容规范

### 标题卡片
- 卡片内顶部：标题文本（加粗，较大字号）
- 卡片内主体：核心内容（1-3 个要点或一段描述）
- 可选：底部标签或来源说明

### 数据卡片
- 中心大号数字
- 数字下方：指标名称
- 可选：趋势箭头（↑↓）和变化百分比

### 内容卡片
- 顶部标题
- 2-4 个要点（每点一行，用 tspan 换行）
- 可选左侧图标占位

## 与金字塔原理的结合

Bento Grid 与金字塔原理的映射：

| 金字塔层级 | Bento Grid 体现 |
|-----------|----------------|
| 核心结论 | 标题卡片或最大卡片 |
| 支撑论点 | 中等卡片（2-3 个并列） |
| 证据/数据 | 数据卡片或小卡片 |
| 背景/补充 | 辅助卡片或图片卡片 |

每页的卡片排列应遵循"结论在前，细节在后"的逻辑。
