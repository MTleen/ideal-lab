# Research Framework — 六维分析框架

本文件定义需求研究的六维分析方法论，指导从源素材中提取结构化的需求信息。

---

## 维度概览

| # | 维度 | 目的 | 核心产出 |
|---|------|------|---------|
| 1 | Message Hierarchy | 明确信息层次 | 核心信息 + 3-5 个支撑论点 + CTA |
| 2 | Audience Decision Matrix | 理解受众决策逻辑 | 受众画像 + 决策障碍 + 证据需求 |
| 3 | Visual Opportunity Map | 识别视觉化机会 | 内容到视觉的映射 + 优先级 |
| 4 | Presentation Flow | 规划叙事结构 | 开场/主体/结尾的模式选择 |
| 5 | Content Adaptation | 筛选内容密度 | 保留/简化/可视化/省略四分法 |
| 6 | Canvas Format Selection | 确定输出格式 | 基于使用场景的画幅决策 |

---

## 维度 1: Message Hierarchy（信息层次）

从源素材中提炼信息的层次结构，确保演示有清晰的核心论点。

### 分析步骤

```
1. 通读源素材，用一句话概括"如果观众只记住一件事，那是什么？"
   → 这就是核心信息 (core_message)

2. 从素材中提取支撑核心信息的论点
   → 筛选 3-5 个最有力度的支撑论点 (supporting_points)
   → 每个论点不超过 15 字

3. 确定行动号召 (Call to Action)
   → 演示结束后，观众应该做什么？
   → 如果源素材没有明确 CTA，根据内容类型推断
```

### 信息层次模板

```
核心信息 (Core Message)
├── 支撑论点 1: {论点} — {来源: 原文段落/数据}
├── 支撑论点 2: {论点} — {来源}
├── 支撑论点 3: {论点} — {来源}
├── 支撑论点 4: {论点} — {来源}（可选）
├── 支撑论点 5: {论点} — {来源}（可选）
└── CTA: {行动号召}
```

### CTA 推断规则

| 内容类型 | 默认 CTA |
|---------|---------|
| 数据分析型 | "基于数据做出某项决策" |
| 技术方案型 | "批准/采纳该技术方案" |
| 战略规划型 | "对齐战略方向，分配资源" |
| 产品发布型 | "试用/购买/推广该产品" |
| 教育培训型 | "掌握某项技能/知识" |
| 故事叙述型 | "产生情感共鸣/改变某种认知" |

## 维度 2: Audience Decision Matrix（受众决策矩阵）

理解受众的立场、信念和决策障碍，为内容策略提供依据。

### 矩阵结构

| 分析维度 | 问题 | 分析方法 |
|---------|------|---------|
| **Who** | 受众是谁？角色、层级、背景 | 从素材语境推断 + 用户确认 |
| **What they believe** | 他们当前的认知/偏见/假设 | 基于内容类型的通用假设 |
| **Desired decision** | 希望他们做出什么决定？ | 从 CTA 反推 |
| **Barriers** | 可能阻碍决策的因素 | 从内容的反对/质疑段落推断 |
| **Evidence needed** | 他们需要什么证据才能做决定？ | 从内容类型推断证据形式 |

### 受众类型快速映射

```
素材中出现的关键词 → 受众类型
├─ "向领导汇报" / "管理层" / "决策层" → 高管 (Executive)
├─ "团队培训" / "入职" / "学习" → 初学者 (Beginner)
├─ "行业分析" / "同行" / "从业者" → 从业者 (Practitioner)
├─ "学术" / "论文" / "研究" → 专家 (Expert)
├─ "客户" / "用户" / "市场" → 混合 (Mixed)
└─ 无法判断 → 从业者 (Practitioner)（默认）
```

## 维度 3: Visual Opportunity Map（视觉化机会图）

识别源素材中适合视觉化呈现的内容，按优先级排序。

### 内容到视觉的映射

| 源素材内容特征 | 推荐视觉形式 | 优先级 |
|--------------|------------|--------|
| 数字/百分比/对比数据 | 柱状图、饼图、数据卡片 | 高 |
| 流程/步骤/时间线 | 流程图、时间轴 | 高 |
| 组织架构/层级关系 | 树状图、层级图 | 中 |
| 对比/优劣势 | 对比表格、双栏布局 | 中 |
| 引用/观点/金句 | 引用卡片、高亮文字 | 中 |
| 案例/故事 | 叙事卡片、图+文组合 | 低 |
| 概念/定义 | 图示、图标+文字 | 低 |
| 纯文字段落 | 需精简为要点或省略 | — |

### 优先级判断

```
优先级 = 视觉影响力 × 内容重要性

视觉影响力：
├─ 数据 → 高（3 分）
├─ 流程/关系 → 高（3 分）
├─ 对比 → 中（2 分）
├─ 观点/引用 → 中（2 分）
├─ 概念 → 低（1 分）
└─ 纯文字 → 极低（0 分）

内容重要性：
├─ 核心信息 → 3 分
├── 支撑论点 → 2 分
└── 补充信息 → 1 分
```

## 维度 4: Presentation Flow（演示流程）

规划演示的叙事结构，选择合适的开场、主体和结尾模式。

### 开场模式 (Opening Patterns)

| 模式 | 说明 | 适用场景 |
|------|------|---------|
| 数据冲击 | 用令人震惊的数据开场 | 数据分析、战略决策 |
| 问题引导 | 提出问题引发思考 | 教育培训、方案汇报 |
| 故事引入 | 用简短故事或案例开场 | 故事叙述、产品发布 |
| 直接陈述 | 直奔主题，陈述核心观点 | 高管汇报、商务提案 |
| 引用名人 | 引用名言或权威观点 | 战略规划、教育培训 |

### 主体模式 (Body Patterns) — 5 种模式

| 模式 | 结构 | 适用场景 |
|------|------|---------|
| **时间线型** | 按时间顺序展开 | 年度报告、项目回顾、发展历程 |
| **问题-方案型** | 提出问题 → 分析 → 解决方案 | 方案汇报、技术咨询 |
| **对比型** | 多个方案/维度的横向对比 | 产品发布、战略选择 |
| **金字塔型** | 从核心观点展开到细节 | 战略决策、管理层汇报 |
| **模块型** | 独立模块并列呈现 | 教育培训、产品功能介绍 |

### 结尾模式 (Closing Patterns)

| 模式 | 说明 | 适用场景 |
|------|------|---------|
| 总结回顾 | 回顾核心要点 | 教育培训、数据分析 |
| 行动号召 | 明确下一步行动 | 方案汇报、战略决策 |
| 愿景展望 | 描绘未来图景 | 战略规划、产品发布 |
| 留悬念 | 提出开放性问题 | 研究报告、学术分享 |
| 致谢收尾 | 简洁致谢 + 联系方式 | 通用 |

## 维度 5: Content Adaptation（内容适配）

对源素材内容进行四分法筛选，决定每部分内容的处理方式。

### 四分法

| 处理方式 | 定义 | 判断标准 |
|---------|------|---------|
| **保留 (Keep)** | 原文保留，直接使用 | 核心论点、关键数据、定义 |
| **简化 (Simplify)** | 压缩篇幅，提炼要点 | 冗长段落、重复论述、背景信息 |
| **可视化 (Visualize)** | 转化为图表/图形 | 数据、流程、对比、关系 |
| **省略 (Omit)** | 不在演示中呈现 | 细节备注、技术细节、参考文献 |

### 判断流程

```
对源素材的每个主要段落/章节：
├── 是否为核心信息或支撑论点？
│   ├── 是 → Keep（保留）
│   └── 否 ↓
├── 是否包含数据/流程/对比？
│   ├── 是 → Visualize（可视化）
│   └── 否 ↓
├── 是否对理解主题有必要？
│   ├── 是 → Simplify（简化后保留）
│   └── 否 → Omit（省略）
```

### 信息密度控制

- 每页幻灯片承载 **1 个核心观点**
- 文字量控制：每页正文不超过 **80 字（中文）** / **60 词（英文）**
- 数据页例外：可承载 **2-3 个相关数据点**

## 维度 6: Canvas Format Selection（画幅格式选择）

基于使用场景确定输出的画幅格式。详见 `canvas-formats.md`。

### 快速决策

```
内容将在哪里使用？
├── 会议/报告/投影 → 演示类 → PPT 16:9 或 PPT 4:3
├── 社交媒体发布 → 社交类 → 按平台选择
├── 打印/纸质 → 印刷类 → A4 纵向
└── 不确定 → PPT 16:9（默认）
```
