# Design Guidelines — 设计原则与指南

本文件定义 PPT 设计的核心原则和指南，涵盖受众分析、视觉层次、配色选择、字体选择和排版范式。

---

## 1. 受众导向设计原则 (Audience-Specific Principles)

### 1.1 初学者受众 (Beginners)

**核心原则**：降低认知负荷，引导注意力

- 每页最多 1 个核心观点
- 使用大量视觉辅助（图标、插图）
- 避免专业术语，必要时添加注释
- 采用渐进式信息呈现
- 步骤型内容使用编号和箭头引导
- 文字精简，每段不超过 3 行

**排版建议**：
- 密度：minimal
- 字号：偏大（基准 24px 以上）
- 留白：≥ 50%

### 1.2 专家受众 (Experts)

**核心原则**：信息密度高，直接切入要点

- 每页可承载 3-5 个要点
- 使用专业术语和数据
- 图表优先，辅以精简文字
- 允许复杂的多维对比
- 提供深层数据和引用来源

**排版建议**：
- 密度：balanced 或 dense
- 字号：标准（基准 18-24px）
- 留白：30-40%

### 1.3 高管受众 (Executives)

**核心原则**：结论先行，数据支撑

- 第一页或每个章节开头即给出结论
- 使用关键指标 (KPI) 和摘要数据
- 图表简洁，去除多余装饰
- 提供决策建议而非单纯数据展示
- 每页一个核心结论 + 3 个支撑论据

**排版建议**：
- 密度：minimal 或 balanced
- 字号：偏大（基准 24px）
- 留白：≥ 40%
- 风格：极简或商务

## 2. 视觉层次 (Visual Hierarchy)

### 2.1 焦点控制 (Focal Point)

每页必须有一个明确的视觉焦点：

1. **主焦点**：页面上最引人注目的元素（最大、最亮、最对比）
2. **次焦点**：支撑主焦点的辅助信息
3. **背景层**：提供上下文但不争夺注意力

焦点建立方法：
- 尺寸差异（2:1 以上）
- 色彩对比（互补色或明暗对比）
- 位置优势（页面上方 1/3 或左侧）
- 留白包围（孤立元素成为焦点）

### 2.2 三分法则 (Rule of Thirds)

将页面分为 3×3 的九宫格：

```
┌─────┬─────┬─────┐
│     │     │     │
│  ★  │     │  ★  │  ← 视觉热点（Power Points）
│     │     │     │
├─────┼─────┼─────┤
│     │     │     │
│     │  ★  │     │  ← 中心点（慎用，可能呆板）
│     │     │     │
├─────┼─────┼─────┤
│     │     │     │
│  ★  │     │  ★  │  ← 视觉热点
│     │     │     │
└─────┴─────┴─────┘
```

- 将关键元素放置在 4 个视觉热点上
- 标题通常位于上方 1/3 线
- 正文内容位于中间 1/3
- 数据/图表位于下方 2/3

### 2.3 Z-Pattern 布局

西方/中文阅读的自然视线路径：

```
Z 路径：
1 ─────────────── 2
                   \
                    \
                     \
3 ─────────────── 4

1: 主标题/Logo（起点）
2: 次要信息/行动点
3: 核心内容区（重点）
4: 结论/CTA（终点）
```

- 沿 Z 路径安排信息重要性递减
- 对角线上放置对比元素可增加动态感
- 数据页面可使用 F-Pattern（从左到右，逐行扫描）

## 3. 内容密度指南 (Content Density Guidelines)

### 3.1 按页面类型

| 页面类型 | 要点数 | 文字上限 | 图表/图片 | 留白比 |
|----------|--------|----------|-----------|--------|
| 封面 | 1 (标题) | 20 字 | 0-1 张 | ≥ 60% |
| 目录 | 全章节 | 50 字 | 0 | ≥ 50% |
| 章节过渡 | 1 (章节名) | 15 字 | 0-1 张 | ≥ 70% |
| 内容讲解 | 3-5 点 | 80 字 | 0-1 张 | ≥ 35% |
| 数据展示 | 2-3 指标 | 40 字 | 1-2 张图表 | ≥ 30% |
| 对比分析 | 2-4 对比项 | 60 字 | 1 张图表 | ≥ 30% |
| 总结 | 3-5 要点 | 60 字 | 0-1 张 | ≥ 40% |
| 结尾 | 1 (CTA) | 30 字 | 0-1 张 | ≥ 60% |

### 3.2 文字密度规则

- 正文每行不超过 35 个中文字符（16:9 画幅）
- 标题不超过 15 个中文字符
- 列表项不超过 20 个中文字符
- 每页总文字量不超过 120 个中文字符（不含标题）

## 4. 配色选择指南 (Color Selection by Content Type)

### 4.1 按内容类型推荐

| 内容类型 | 推荐色系 | 理由 |
|----------|----------|------|
| 财务数据 | Navy Blue + Gold | 稳重、专业、价值感 |
| 技术架构 | Tech Blue + Cyan | 科技、逻辑、清晰 |
| 市场分析 | Orange + Blue | 活力、洞察、对比 |
| 战略规划 | Deep Blue + Accent | 权威、远见、决策 |
| 产品介绍 | 品牌色系 | 一致性、识别度 |
| 教育培训 | Blue-Green + Warm | 平和、知识、成长 |
| 创意提案 | Purple + Warm | 创意、独特、吸引 |
| 进度汇报 | 绿/蓝 + 数据色 | 清晰、客观、进度 |

### 4.2 色彩心理学参考

| 颜色 | 心理联想 | 适用场景 |
|------|----------|----------|
| 蓝色 | 信任、专业、冷静 | 企业、科技、金融 |
| 绿色 | 成长、健康、安全 | 增长数据、环保、医疗 |
| 橙色 | 活力、创新、热情 | 创意、营销、推广 |
| 红色 | 紧迫、重要、力量 | 警示、关键数据、政府 |
| 紫色 | 创意、高端、神秘 | 品牌、创意、奢侈品 |
| 黄色 | 积极、温暖、关注 | 高亮、重点、友好 |

## 5. 字体原则与推荐 (Typography Principles & Recommendations)

### 5.1 字体选择原则

1. **可读性优先**：正文必须选择高可读性的无衬线字体
2. **层级分明**：标题和正文使用不同字重，而非不同字体
3. **中英协调**：中英文字体的视觉重心和 x-height 需匹配
4. **品牌一致**：如有品牌字体规范，优先遵守
5. **数量控制**：单个项目使用的字体族不超过 2 个

### 5.2 中文字体推荐

| 字体名称 | 风格分类 | 适用场景 | 特点 |
|----------|----------|----------|------|
| **MiSans** | 无衬线/几何 | 通用、标题、正文 | 现代、均衡、小米品牌字体 |
| **Noto Sans SC** | 无衬线/人文 | 正文、长文本 | Google 出品、覆盖全字符集 |
| **思源宋体 (Source Han Serif)** | 衬线 | 标题、引用、论文 | Adobe+Google 联合、典雅 |
| **阿里巴巴普惠体** | 无衬线/粗犷 | 标题、大字展示 | 粗壮有力、视觉冲击 |
| **霞鹜文楷 (LXGW Bright)** | 手写/楷体 | 引用、文艺、教育 | 温和、人文感、独特 |

### 5.3 英文字体推荐

| 字体名称 | 风格分类 | 适用场景 | 特点 |
|----------|----------|----------|------|
| **Liter** | 几何无衬线 | 标题、展示 | 独特、现代、辨识度高 |
| **HedvigLettersSans** | 人文无衬线 | 正文、长文本 | 温暖、可读性好 |
| **Oranienbaum** | 衬线/古典 | 标题、引用 | 优雅、高端、对比强 |
| **Coda** | 等宽/技术 | 数据、代码、技术 | 清晰、技术感 |

### 5.4 字体搭配方案

**方案 P1 — 现代商务**：
```
中文标题: MiSans Bold (700)
中文正文: MiSans Regular (400)
英文标题: Liter Bold (700)
英文正文: HedvigLettersSans Regular (400)
```

**方案 P2 — 科技专业**：
```
中文标题: MiSans SemiBold (600)
中文正文: Noto Sans SC Regular (400)
英文标题: Coda Bold (700)
英文正文: HedvigLettersSans Regular (400)
```

**方案 P3 — 学术论文**：
```
中文标题: 思源宋体 Bold (700)
中文正文: 思源宋体 Regular (400)
英文标题: Oranienbaum Regular (400)
英文正文: HedvigLettersSans Regular (400)
```

**方案 P4 — 创意人文**：
```
中文标题: 阿里巴巴普惠体 Bold (700)
中文正文: 霞鹜文楷 Regular (400)
英文标题: Liter Bold (700)
英文正文: HedvigLettersSans Light (300)
```

**方案 P5 — 极简高端**：
```
中文标题: MiSans Light (300)
中文正文: MiSans Regular (400)
英文标题: Oranienbaum Regular (400)
英文正文: HedvigLettersSans Light (300)
```

## 6. Bento Grid 排版范式 (Primary Layout Paradigm)

### 6.1 核心理念

Bento Grid（便当盒网格）借鉴日式便当盒的分区理念：

- 每个信息单元是一个独立的"格子"
- 格子大小不等，反映信息权重
- 格子之间有均匀间距
- 整体呈现整齐有序的视觉节奏

### 6.2 基础网格定义

```
基础网格：12 列
行高：自适应（基于内容）
间距 (Gap)：16px 或 24px
卡片圆角：8px 或 12px
卡片内边距：24px 或 32px
```

### 6.3 常用 Bento 布局模板

**模板 A — 2 区域**：
```
┌─────────────────────┐
│                     │
│     主内容区域       │
│     (8 列)          │
│                     │
├──────────┬──────────┤
│  辅助 A  │  辅助 B  │
│  (4 列)  │  (4 列)  │
└──────────┴──────────┘
```

**模板 B — 3 区域**：
```
┌──────────┬──────────┐
│          │          │
│  主内容   │  侧边栏  │
│  (8 列)  │  (4 列)  │
│          │          │
├──────────┴──────────┤
│    底部信息栏        │
│    (12 列)          │
└─────────────────────┘
```

**模板 C — 4 区域（经典便当）**：
```
┌──────────┬─────┬─────┐
│          │ B1  │ B2  │
│  A (大)  ├─────┴─────┤
│          │    C      │
├──────────┴───────────┤
│         D           │
└─────────────────────┘
```

**模板 D — 仪表盘**：
```
┌─────┬─────┬─────┬─────┐
│ KPI │ KPI │ KPI │ KPI │
├─────┴─────┼─────┴─────┤
│           │           │
│  图表 A   │  图表 B   │
│           │           │
├───────────┴───────────┤
│      明细数据表格      │
└───────────────────────┘
```

### 6.4 Bento Grid 设计规则

1. **对齐**：所有卡片边缘必须对齐到网格线
2. **间距一致**：所有卡片间距保持统一（Gap 值）
3. **圆角统一**：所有卡片使用相同的 border-radius
4. **层级区分**：通过背景色深浅区分卡片层级
5. **留白呼吸**：卡片内边距 ≥ Gap 值
6. **焦点突出**：最大卡片放置最重要信息
7. **视觉平衡**：大卡片和小卡片交替排列，避免单调
8. **响应内容**：卡片高度自适应内容，不强求等高

## 7. 设计质量检查要点

### 视觉一致性
- 配色方案在所有页面一致
- 字体和字号层级不偏离定义
- 间距和圆角保持统一
- 图标风格一致

### 可读性
- 正文对比度 ≥ 4.5:1
- 每行字符数不超限
- 文字不被图形遮挡
- 中英文混排美观

### 专业度
- 无拼写错误
- 数据标注来源
- 无占位内容
- 视觉语言统一
