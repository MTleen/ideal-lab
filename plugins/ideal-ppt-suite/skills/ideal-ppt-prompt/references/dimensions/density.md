# 密度维度

单页幻灯片的信息密度。

## 选项

| 选项 | 内容/页 | 留白 | 适用场景 |
|------|---------|------|----------|
| `minimal` | 单一焦点 | 最大 | 高管汇报、 keynote、情感冲击 |
| `balanced` | 2-3 个要点 | 标准 | 通用演示、混合受众 |
| `dense` | 多个数据点 | 紧凑 | 数据密集型、技术评审、详细分析 |

## 渲染指南

### minimal

- 每页 ONE 主信息
- 大视觉主导画面
- 最少文字（标题 + 最多 1-2 行）
- 宽裕边距（距边缘 15%+）
- 元素之间最大化呼吸空间
- 让单一元素承载全部视觉重量

**原则**：
- "一页一信息"
- 视觉 > 文字
- 空白是刻意的
- 每个元素都必须有其存在的理由

### balanced

- 每页 2-3 个要点
- 标准边距（距边缘 10%）
- 文字/视觉比例均衡
- 清晰的层级与支撑细节
- 舒适的阅读体验

**原则**：
- 主信息 + 支撑上下文
- 视觉辅助文字
- 结构化但不拥挤
- 适合多元受众

### dense

- 允许多个数据点
- 紧凑边距（距边缘 5-8%）
- 信息丰富的布局
- 图表、表格、详细注释
- 假设受众专注、注意力集中

**原则**：
- 数据比装饰更有说服力
- 注释解释洞察
- 留白是策略性的
- 每个像素都有用途

## 受众 → 密度映射

| 受众 | 推荐密度 |
|------|---------|
| 高管 | minimal |
| 初学者 | minimal 至 balanced |
| 普通受众 | balanced |
| 中级受众 | balanced |
| 专家 | balanced 至 dense |

## 幻灯片类型 → 密度指南

| 幻灯片类型 | 推荐密度 |
|------------|---------|
| 封面/标题 | minimal |
| 章节分隔 | minimal |
| 引用/冲击 | minimal |
| 议程/概览 | balanced |
| 内容/分析 | balanced 或 dense |
| 总结/要点 | balanced |
| 数据/指标 | dense |

## 各密度下的内容指南

### minimal

| 元素 | 指南 |
|------|------|
| 标题 | 大号（相当于 40-60pt） |
| 正文 | 最少或无 |
| 项目符号 | 最多 0-2 个 |
| 视觉元素 | 1 个主导元素 |
| 图表/数据 | 仅 1 个关键数据 |

### balanced

| 元素 | 指南 |
|------|------|
| 标题 | 中大号（相当于 32-48pt） |
| 正文 | 2-4 行 |
| 项目符号 | 2-4 个 |
| 视觉元素 | 1-2 个元素 |
| 图表/数据 | 简单图表可用 |

### dense

| 元素 | 指南 |
|------|------|
| 标题 | 中号（相当于 24-36pt） |
| 正文 | 允许多段 |
| 项目符号 | 4-6+ 个 |
| 视觉元素 | 允许多个 |
| 图表/数据 | 复杂图表、表格可用 |

## 组合注意事项

| 密度 | 最佳搭配 | 避免搭配 |
|------|----------|----------|
| minimal | neutral 色调, geometric 字体 | 密集数据内容 |
| balanced | 任意色调/字体 | 极端（过疏或过密） |
| dense | cool 色调, technical 字体 | handwritten 字体, organic 纹理 |
