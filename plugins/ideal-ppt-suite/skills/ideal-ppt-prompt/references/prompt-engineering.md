# Prompt Engineering Guide — SVG 提示词编写指南

本文档定义将幻灯片大纲转换为自包含 SVG 生成提示词的完整规范。

---

## 提示词结构

每个提示词文件由以下三部分组成：

```
┌─────────────────────────────┐
│  Part 1: 元数据 (Metadata)   │  幻灯片编号、类型、目标文件名
├─────────────────────────────┤
│  Part 2: STYLE_INSTRUCTIONS │  风格参数（从 design-spec.md 提取）
├─────────────────────────────┤
│  Part 3: SLIDE CONTENT      │  具体内容和视觉布局描述
└─────────────────────────────┘
```

### Part 1: 元数据

每个提示词文件头部必须包含以下元数据块：

```markdown
---
slide_number: 03
slide_type: data           # cover | section | content | data | comparison | process | quote | ending
target_filename: 03-slide-market-data.md
headline: "市场规模达 6,800 亿元，年增速 35%"
---
```

### Part 2: STYLE_INSTRUCTIONS

从 `design-spec.md` 提取的完整视觉参数。必须包含：

| 参数 | 说明 | 示例 |
|------|------|------|
| 背景色 | 页面背景颜色 | `#FFFFFF` |
| 主色 (Primary) | 大面积色块、关键图形 | `#005587` |
| 辅助色 (Secondary) | 标题、次要图形 | `#2196F3` |
| 强调色 (Accent) | 高亮、标注、数据重点 | `#FF9800` |
| 文字色 | 主文字颜色 | `#1A1A2E` |
| 次要文字色 | 说明文字、标签 | `#666666` |
| 标题字体栈 | font-family 声明 | `'PingFang SC', 'Microsoft YaHei', sans-serif` |
| 正文字体栈 | font-family 声明 | 同上 |
| 标题字号 | 主标题字号 (px) | `36` |
| 副标题字号 | 副标题字号 (px) | `24` |
| 正文字号 | 正文内容字号 (px) | `16` |
| 标注字号 | 小标签、注释字号 (px) | `12` |

### Part 3: SLIDE CONTENT

具体内容和视觉描述。这是提示词的核心，必须做到自包含。

---

## 内容完整性规则

### 规则 1: 列表必须有具体内容

列表项不能只包含标签，必须附带具体描述或数据。

| 类型 | 示例 |
|------|------|
| BAD | "三个核心优势" |
| BAD | "技术优势：速度、安全、成本" |
| GOOD | "3 列优势卡片：左列 '响应速度 < 50ms，P99 延迟低于行业平均 3 倍'，中列 'SOC 2 Type II 认证，数据加密存储 + 传输'，右列 'TCO 降低 40%，按需付费无隐藏费用'" |

### 规则 2: 机制/流程必须有步骤

涉及流程、机制、架构的描述，必须给出每一环节的具体名称和动作。

| 类型 | 示例 |
|------|------|
| BAD | "数据处理流程图" |
| BAD | "展示从输入到输出的完整流程" |
| GOOD | "水平 4 步流程箭头：Step 1 '数据采集'（蓝色方块，API + SDK 多源接入）→ Step 2 '清洗转换'（绿色方块，去重 + 格式化 + 特征提取）→ Step 3 '模型推理'（橙色方块，在线推理引擎 + A/B 路由）→ Step 4 '结果输出'（紫色方块，REST API + WebSocket 双通道推送）" |

### 规则 3: 数据必须有实际数值

所有数据展示必须包含具体数值和单位。

| 类型 | 示例 |
|------|------|
| BAD | "市场份额持续增长" |
| BAD | "显示增长趋势的数据图表" |
| GOOD | "柱状图：X 轴为年份（2022/2023/2024/2025），Y 轴为市场规模（亿元）。数据：2022 年 3,200 亿、2023 年 4,500 亿、2024 年 5,800 亿、2025 年 6,800 亿（预测值标注 *）。柱体使用 #2196F3 填充，2025 柱体使用 #FF9800 强调" |

### 规则 4: 对比必须有维度和内容

任何对比展示必须列出对比维度和每项的具体内容。

| 类型 | 示例 |
|------|------|
| BAD | "新旧方案对比表" |
| BAD | "展示两种方案的差异" |
| GOOD | "2 列对比布局：左列 '传统方案'（灰色边框卡片）— 部署周期 4-6 周 / 运维人力 5 人+ / 扩展方式 手动 / 月成本 ¥8 万+。右列 '我们的方案'（蓝色边框卡片）— 部署周期 3 天 / 运维人力 1 人 / 扩展方式 自动弹性 / 月成本 ¥2.5 万。每行使用 ✓/✗ 图标标注" |

## 视觉描述规则

提示词中的视觉描述必须描述**具体的视觉元素**，而非抽象概念。

### 描述粒度要求

| 元素类型 | 必须包含的信息 |
|---------|--------------|
| 文本 | 具体文字内容、字号、颜色、对齐方式、大致位置 |
| 色块/卡片 | 颜色（HEX）、尺寸比例、圆角、边框、内部文字 |
| 图表 | 图表类型、坐标轴数据、颜色编码、标注 |
| 布局 | 元素排列方式（行列数）、间距、对齐、分区 |
| 连线/箭头 | 起止元素、方向、颜色、标注文字 |

### 正确的视觉描述示例

```
页面布局：上下两区域，比例 3:7

上半部分（y: 0-200）：
  - 主标题 "市场规模达 6,800 亿元"，字号 36px，颜色 #1A1A2E，
    左对齐，位置 (60, 80)
  - 副标题 "中国云计算市场 2022-2025 年增长趋势"，字号 18px，
    颜色 #666666，左对齐，位置 (60, 130)

下半部分（y: 200-720）：
  - 4 根垂直柱状图，等间距排列
  - 每根柱宽 120px，间距 80px，起始 x: 180
  - 数据从左到右：3,200 / 4,500 / 5,800 / 6,800*
  - 柱体颜色 #2196F3，最后一根 #FF9800
  - X 轴标签：2022 / 2023 / 2024 / 2025*
  - Y 轴范围 0-8,000，标注 "单位：亿元"
```

### 错误的视觉描述示例

```
页面展示市场增长数据
用柱状图呈现
颜色使用品牌色
适当标注数据
```

## 提示词长度指南

| 幻灯片类型 | 建议提示词长度 | 说明 |
|-----------|--------------|------|
| 封面 (cover) | 50-80 行 | 布局简洁，但文字内容必须具体 |
| 章节过渡 (section) | 30-50 行 | 通常只有标题和简短描述 |
| 内容页 (content) | 80-150 行 | 最常见类型，需要详细的布局描述 |
| 数据页 (data) | 100-180 行 | 图表需要完整的数据和样式描述 |
| 对比页 (comparison) | 80-150 行 | 列出所有对比维度和内容 |
| 流程页 (process) | 80-150 行 | 每个步骤的具体描述和连接方式 |
| 引用页 (quote) | 40-60 行 | 内容较少，布局描述简短 |
| 封底 (ending) | 50-80 行 | 行动号召或总结，内容需具体 |

## 幻灯片类型与提示词模板

### cover 类型

```
元数据 + STYLE_INSTRUCTIONS +
SLIDE CONTENT:
  - 主标题文字（具体内容）
  - 副标题文字（具体内容）
  - 日期/署名（具体内容）
  - 视觉元素描述（背景装饰、图形）
  - 布局（居中/左对齐，具体位置）
```

### data 类型

```
元数据 + STYLE_INSTRUCTIONS +
SLIDE CONTENT:
  - 标题（叙事性标题）
  - 图表类型声明（柱状图/折线图/饼图/数字卡片）
  - 完整数据集（每项有具体数值）
  - 图表样式（颜色、标注、坐标轴）
  - 辅助说明文字
  - 布局（图表区域位置和尺寸）
```

### comparison 类型

```
元数据 + STYLE_INSTRUCTIONS +
SLIDE CONTENT:
  - 标题
  - 对比对象名称
  - 对比维度列表（每个维度有具体内容）
  - 每个对象在每个维度的具体值
  - 视觉高亮规则（哪个对象在哪些维度有优势）
  - 布局（左右分栏 / 表格 / 卡片）
```
