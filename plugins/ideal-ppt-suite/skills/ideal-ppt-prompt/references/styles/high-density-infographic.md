# 高密度信息图风格 (High-Density Infographic)

## 概述

专业咨询风格 + 有意义的信息图表。核心理念：

- **内容驱动密度**：信息密度来自实际文字和数据，不是视觉装饰
- **有意义的图表**：流程图、对比图、趋势图等辅助理解，但禁止装饰性插图
- **干净排版**：纯白背景，无网格/纹理，用色块和分隔线组织结构
- **小字体高密度**：较小字号塞更多内容，而非大字少内容

## 视觉特征

### 布局原则
- 16:9 宽屏比例
- 纯白背景 (#FFFFFF)，无任何背景纹理或网格线
- 适度留白（边距 8-10%），内容充实但不拥挤
- 标题占用空间小，内容区域最大化

### 鼓励使用的视觉元素

#### 1. 流程图 (Flowcharts)
展示实际步骤和流转关系：
```
┌──────┐    ┌──────┐    ┌──────┐    ┌──────┐
│ 阶段1 │───→│ 阶段2 │───→│ 阶段3 │───→│ 输出  │
└──────┘    └──────┘    └──────┘    └──────┘
```
- 节点内必须有文字标签
- 箭头简洁（直线+箭头，不要花哨样式）

#### 2. 对比布局
左右/上下对比结构，用颜色区分：
```
┌─────────────────┬─────────────────┐
│  传统方式 (灰)  │  新方式 (蓝)    │
├─────────────────┼─────────────────┤
│ 详细数据/说明   │ 详细数据/说明    │
│ • 要点1         │ • 要点1         │
│ • 要点2         │ • 要点2         │
└─────────────────┴─────────────────┘
```

#### 3. 数据图表
用于展示量化数据：
- 柱状图（对比数值）
- 趋势线（展示变化）
- 饼图（展示占比）
- 图表中必须有数据标签

#### 4. 结构化表格
展示多维度信息：
```
┌──────────┬──────────┬──────────┐
│ 维度     │ 指标     │ 说明     │
├──────────┼──────────┼──────────┤
│ 内容     │ 内容     │ 内容     │
└──────────┴──────────┴──────────┘
```

#### 5. 层级/树状图
展示组织结构或分类关系

#### 6. 色块卡片
带彩色顶部标题栏的内容卡片：
```
┌─────────────────────────────────────┐
│ ▓▓▓ 标题栏 (蓝色背景白字) ▓▓▓▓▓▓  │
├─────────────────────────────────────┤
│  具体内容文字                       │
│  - 要点说明                         │
│  - 数据展示                         │
└─────────────────────────────────────┘
```

### 严格禁止的视觉元素

- **装饰性图标/clipart**：时钟、大脑、灯泡、齿轮、人物剪影等
- **背景网格线/蓝图纹理**：任何背景装饰图案
- **伪代码装饰**：随机的代码片段、哈希编号、变量名等作为填充
- **抽象插画**：与内容无关的形状、波浪线、圆点装饰
- **重复展示**：同一数据用不同形式重复呈现

## 色彩系统

### 主色调
- Primary: 深藏青 (#1E3A5F) — 标题、重要文字
- Accent: 工程蓝 (#2563EB) — 高亮、关键数据、章节头
- Success: 绿色 (#10B981) — 正面/成功指标
- Warning: 橙色 (#F59E0B) — 警告
- Danger: 红色 (#EF4444) — 问题/风险

### 背景色
- Card Background: #FFFFFF (白)
- Alternate Row / Card Fill: #F0F4F8 (浅灰蓝)
- Positive Indicator: #E8F4E8 (浅绿)
- Negative Indicator: #FEE8E8 (浅红)

### 文字色
- Primary Text: #1E3A5F (深藏青)
- Secondary Text: #64748B (灰色)
- White Text: #FFFFFF (用于深色背景上)

## 排版规范

### 字体
- 标题: Bold Sans-Serif (SimHei / Microsoft YaHei)
- 正文: Clean Sans-Serif
- 行距: 1.2-1.4（紧凑）

### 字号层级（偏小，容纳更多内容）
- 幻灯片标题: 28-32pt（不要更大）
- 卡片/小节标题: 16-18pt
- 正文: 12-14pt
- 数据标签/注释: 10-12pt

## 内容密度指南

### 密度原则
1. **每页 6-8 个要点** 或等量的图表+文字组合
2. **图文混排**：文字块 + 图表在同一页共存
3. **小标题大内容**：标题区精简，把空间留给内容
4. **表格优于段落**：结构化信息用表格展示
5. **箭头优于文字**：用 `→` 替代 "然后"、"接着"

### 内容填充策略
- 空间不够 → 缩小字号，不要删减内容
- 空间太多 → 增加有意义的图表或补充数据，**不要用装饰填充**
- 如果确实内容少 → 留白好过用图标凑数

## 布局变体

### 封面布局
- 深色背景 (#1E3A5F)
- 标题居中，副标题在下
- 底部版本信息小字
- 简洁，不需要装饰

### 目录布局
- 编号列表，紧凑排列
- 可用色块分组
- 左对齐

### 内容布局（最常用）
- 2-3 列并排卡片
- 或：左侧文字 + 右侧图表
- 或：上方标题 + 下方表格/流程图

### 对比布局
- 左右分栏，颜色区分
- Before/After 或 传统/新方式
- 用数据说话

### 流程布局
- 横向阶段流（3-6个节点）
- 节点内有详细文字
- 输入 → 处理 → 输出

### 封底布局
- 居中总结信息
- 简洁收束

## 适用场景

- 技术产品演示
- 技术方案汇报
- 架构说明
- 工作流文档
- 竞争优势对比
- 端到端流程展示

## 不适用场景

- 需要大量解释性文字的教学课件
- 面向非技术人员的简洁汇报
- 社交媒体内容
