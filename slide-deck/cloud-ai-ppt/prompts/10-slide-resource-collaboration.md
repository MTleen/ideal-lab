# PPT Prompt - Slide 10: Resource Collaboration

=== Style Brief ===
Visual style: Professional technology presentation, blue-themed, clean card-based layout
Color palette: Primary #1A56DB(telecom blue) / Secondary #7C3AED(tech purple) / Accent #F59E0B(data orange)
Module colors: Red=#DC2626(core/AI) Blue=#2563EB(cloud) Orange=#D97706(users) Green=#059669(data)
Typography: Clear hierarchy - titles bold/dark, body readable, numbers large
Font: PingFang SC, Microsoft YaHei, Noto Sans SC, sans-serif
Density: dense - pack data, use tables/cards/architecture diagrams, minimize whitespace
Canvas: 1280x720 (16:9)
Language: Chinese

=== Slide Type ===
Content slide - three-component collaboration diagram

=== Content ===

标题（顶部，粗体）：
计算+存储+网络——协同管理

三大组件（中央区域，使用三角形或三列布局，每个组件使用不同模块色）：

组件1 - 计算（使用红色 #DC2626）
- 图标：👨‍🍳 厨师
- 角色：训练和运行AI模型
- 职责：
  - 模型训练：反复学习数据中的规律
  - 模型推理：快速给出预测结果
  - 性能指标：GPU算力、并行处理能力

组件2 - 存储（使用蓝色 #2563EB）
- 图标：🏭 仓库
- 角色：安全存放海量数据
- 职责：
  - 数据保管：安全存储、防止丢失
  - 快速读取：训练时高速传输数据
  - 数据管理：版本控制、权限管理

组件3 - 网络（使用橙色 #D97706）
- 图标：🚚 运输
- 角色：高速数据传输通道
- 职责：
  - 数据传输：在计算和存储之间快速搬运
  - 低延迟：确保数据及时到达
  - 高带宽：支持海量数据同时传输

协同关系图（三个组件之间用箭头连接，形成三角关系）：
- 计算 ↔ 存储：模型训练时需要大量数据读取
- 存储 ↔ 网络：数据通过网络安全传输
- 网络 ↔ 计算：高速网络确保算力不浪费

核心原则（底部醒目区域，使用绿色 #059669 或主色调）：
🔑 核心原则：算力跟着数据走
- 数据在哪里，计算就在哪里执行
- 减少数据搬运，提升整体效率
- 三者协同，缺一不可

=== Layout Guidance ===
- 三大组件使用三角形或三列布局，每个组件独立卡片
- 组件之间用双向箭头连接，标注协同关系
- 每个组件包含：图标、角色比喻、具体职责
- 底部核心原则使用醒目样式，"算力跟着数据走"特别强调
- 类比（厨师/仓库/运输）与正式描述并列，辅助理解
- 整体信息密度高，通过色彩分组和连线保持清晰
