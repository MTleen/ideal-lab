## Headline
第三层：精装方案包——MaaS

## Subtitle / Context
MaaS = Model as a Service（模型即服务）。这是云智融合四层架构的第三层。如果说前两层是毛坯房里的基础设施和施工工具，第三层就是"精装方案包"——AI模型已经训练好、打包好，你只需要选择、微调、直接使用。

## Core Content

### MaaS 三步走

**第一步：选菜谱（Choose）**
- 浏览数百个预训练AI模型
- 涵盖自然语言处理、计算机视觉、语音识别、多模态等各领域
- 像在美食平台上浏览菜谱——川菜、粤菜、日料、西餐，应有尽有

**第二步：改菜谱（Customize）**
- 用你自己的数据对模型进行微调优化
- 让通用模型变成你的专属模型
- 就像告诉厨师："我要少油少盐，多加辣"——个性化定制

**第三步：上桌吃（Deploy）**
- 调用API接口，直接使用
- 无需关心模型部署、服务器运维、性能调优
- 像点外卖一样简单——下单即送达

### MaaS类比：AI的外卖平台
- 传统方式 = 自己买菜、学做菜、洗锅碗 — 从零训练模型
- MaaS = 打开外卖平台、选餐厅、下单 — 选择模型、调用接口
- 不需要知道菜是怎么做的，只需要知道好不好吃
- 不需要知道模型怎么训练的，只需要知道好不好用

### 在四层架构中的位置
- 第一层（地基）：基础设施——算力/存储/网络
- 第二层（工具）：平台工具——AI开发/训练/部署全套
- **第三层（精装）：AI服务——MaaS模型即服务** ← 本页
- 第四层（入住）：应用——智能客服/语音助手/图像识别

## Key Insights
  - 洞察 1: MaaS是AI民主化的关键 — 中小企业不再需要从零训练，直接使用预训练模型
  - 洞察 2: 三步走流程极简 — 选/改/用，把AI开发周期从"年"缩短到"周"
  - 洞察 3: "外卖平台"类比直观 — 不需要会做饭也能吃好饭，不需要会训练模型也能用AI
