# PPT Prompt - Slide 06: Self-Build Too Expensive

=== Style Brief ===
Visual style: Professional technology presentation, blue-themed, clean card-based layout
Color palette: Primary #1A56DB(telecom blue) / Secondary #7C3AED(tech purple) / Accent #F59E0B(data orange)
Module colors: Red=#DC2626(core/AI) Blue=#2563EB(cloud) Orange=#D97706(users) Green=#059669(data)
Typography: Clear hierarchy - titles bold/dark, body readable, numbers large
Font: PingFang SC, Microsoft YaHei, Noto Sans SC, sans-serif
Density: dense - pack data, use tables/cards/architecture diagrams, minimize whitespace
Canvas: 1280x720 (16:9)
Language: Chinese

=== Slide Type ===
Content slide - cost breakdown with emphasis on impracticality

=== Content ===

标题（顶部，粗体）：
自己建"厨房"？经济上行不通

成本拆解（中央区域，使用图标+数字卡片或表格）：

项目1 - AI芯片（使用红色 #DC2626 标注）
- 💰 一张AI芯片 = 一辆车的价格
- 数量需求：数百到数千张
- 图标：芯片图标 🖥️

项目2 - 配套设施（使用蓝色 #2563EB 标注）
- 服务器 + 存储 + 散热系统
- 总投资：数亿到数十亿
- 图标：服务器机房图标 🏢

项目3 - 持续成本（使用橙色 #D97706 标注）
- ⚡ 90%的时间设备闲置
- 但电费 + 维护费用照常支出
- 图标：电费/维护图标 🔧

关键问题（右侧或下方醒目区域）：
🔄 技术更新周期：仅2年
- 2年后新一代芯片发布
- 旧设备性能落后
- 需要重新投入巨资

底部核心结论（粗体大字，使用红色 #DC2626 强调）：
❌ 只有巨头企业才能承担如此成本
普通人/中小企业 → 自建算力 = 不现实

=== Layout Guidance ===
- 成本拆解使用卡片或表格形式，每个项目独立一块
- 数字（一辆车、数亿、90%、2年）用大号字体突出
- 底部结论使用强调色和较大字号
- 可在角落加入"天平"图标暗示成本效益对比
- 信息密集但分组清晰，视觉上有层次感
