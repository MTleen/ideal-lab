# 03 - 平台架构

[Transition] 了解了定位，我们来看看 Harness 的技术架构是如何设计的。

Harness 采用四层分离架构：最上层是接入网关，提供 API Gateway、GraphQL、Webhook 等多种接入方式。中间是平台编排层，Pipeline Engine 是核心，负责整个交付流程的编排和调度。再往下是执行引擎层，包含 CI Builder、CD Deployer、STO Scanner 等六大执行器。最底层是基础设施层，通过 Delegate 代理模式连接各种云和 K8s 集群。

**要点：**
① 四层分离：接入→编排→执行→基础设施
② Delegate 代理：出站连接，安全可靠
③ AIDA AI 引擎贯穿全平台

**时长：** 75s
