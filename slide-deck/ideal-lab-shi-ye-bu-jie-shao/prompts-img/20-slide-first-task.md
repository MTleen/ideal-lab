Title: "第一个任务：完整体验"

Content — walk through your first task using document-workflow:

Launch:
`/ideal-document-workflow:ideal-document-workflow 帮我写一份"团队代码评审规范"介绍文档，目标读者是事业部开发团队，篇幅 1500 字左右`

Phase 1 — 需求确认:
工作流分析你的需求，确认关键点：
  - 文档主题：团队代码评审规范
  - 目标读者：事业部开发团队
  - 篇幅要求：约 1500 字
  - 写作风格：实用导向
回复 `y` 确认进入下一阶段

Phase 2 — 大纲审核:
工作流生成大纲，你审核章节结构：
  1. 为什么要做代码评审
  2. 评审流程（PR 提交 → 评审 → 修改 → 合并）
  3. 评审检查清单
  4. 常见问题与最佳实践
回复 `y` 或提出修改意见

Phase 3 — 等待写作:
工作流自动并行写作各章节，期间不需要操作

Phase 4 — 获得交付物:
输出最终文档，保存在指定路径

Time: 整个过程约 5-10 分钟，你只需在需求确认和大纲审核两个节点做决策
