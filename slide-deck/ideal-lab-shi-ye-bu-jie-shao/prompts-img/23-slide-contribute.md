Title: "反馈与共建"

Content — three ways to engage with ideal-lab:

1. 问题反馈:
   通过 GitHub Issue 描述你遇到的问题
   包括：使用场景、期望行为、实际行为、复现步骤

2. Skill 共建:
   如果你在日常工作中沉淀出了一套好用的流程，可以按照 ideal-lab 的 Plugin 结构封装为新的 Skill
   通过 PR 提交，经过审核后合入主仓库
   封装规范参考 ideal-lab-maintainer 的 add-skill 命令

3. 经验分享:
   使用 ideal-lab 产出的文档、PPT、调研报告，本身就是最好的经验沉淀
   分享你的使用心得和改进建议

Key message: "ideal-lab 是持续迭代的活系统。使用过程中发现更好的做法，就是下一次迭代的起点"
