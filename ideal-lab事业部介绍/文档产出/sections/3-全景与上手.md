## 4. 能力全景：9 大插件一览

ideal-lab 目前包含 9 个插件、42 个 Skill。它们按用途分为三类：四大工作流套件、三大独立工具、两大开发辅助。

### 4.1 四大工作流套件

工作流套件是 ideal-lab 的核心。每个套件都是一个"编排器 + Phase Skill"的组合——编排器负责调度流程，Phase Skill 负责执行具体阶段。这意味着你可以调用编排器一键走完全流程，也可以单独调用某个 Phase Skill 精细化操作。

| 插件 | 一句话定位 | 核心阶段 | 适用场景 | 安装命令 |
|------|-----------|---------|---------|---------|
| **ideal-dev-workflow** (v1.1.0, 14 Skills) | 15 阶段开发流程：从需求到交付 | 需求 → 技术方案 → 编码计划 → 测试用例 → 开发执行 → 代码评审 → 测试执行 → 维基更新 → 成果提交 | 功能开发、Bug 修复、重构任务 | `claude plugin install ideal-dev-workflow@ideal-lab` |
| **ideal-ppt-suite** (v1.0.0, 9 Skills) | PPT 全链路生成，17 种预设风格 | 调研 → 策略（八确认） → 大纲（金字塔原理） → 提示词工程 → 生成 → AI 配图 → PPTX 导出 | 方案汇报、产品路演、培训课件 | `claude plugin install ideal-ppt-suite@ideal-lab` |
| **ideal-document-workflow** (v1.0.0, 8 Skills) | 12 阶段结构化文档写作 | 需求分析 → 大纲 → 任务拆分 → 并行写作 → 智能配图 → 渲染输出 | 技术文档、方案文档、知识沉淀 | `claude plugin install ideal-document-workflow@ideal-lab` |
| **ideal-knowledge-base** (v1.0.0, 6 Skills) | 知识库构建编排 | 材料分析 → 文档规划 → 并行生成 → 汇总评审 | 项目知识沉淀、团队 Wiki 构建、技术积累 | `claude plugin install ideal-knowledge-base@ideal-lab` |

四个套件共享相同的设计模式：手动模式下每阶段确认后推进，YOLO 模式下自动运行直到完成或触发熔断。前文已经用 ideal-document-workflow 做了详细拆解，另外三个套件的运行机制完全一致。

### 4.2 三大独立工具

**ideal-deep-research** (v1.0.0) — 企业级深度调研。提供 4 种深度模式：`quick`（2-5 分钟快速扫描）、`standard`（5-10 分钟标准调研，默认）、`deep`（10-20 分钟深度分析）、`ultradeep`（20-45 分钟全面审查）。调研管道包含 SCOPE → PLAN → RETRIEVE → TRIANGULATE → OUTLINE → SYNTHESIZE → CRITIQUE → REFINE → PACKAGE 共 8 个阶段，要求覆盖 10+ 信息源、每条结论至少 3 条引用、对信息源做可信度评分。适用于竞品分析、技术选型、趋势调研等需要多源交叉验证的场景。

```bash
claude plugin install ideal-deep-research@ideal-lab
# 调用：/ideal-deep-research:deep-research
```

**baoyu-image-gen** (v1.0.0) — 多 Provider AI 图片生成工具。支持 OpenAI (DALL-E)、Google (Imagen) 和 DashScope (通义万相) 三个图片生成 Provider，可以按需切换。适用于文档配图、PPT 插图、营销素材等场景。

```bash
claude plugin install baoyu-image-gen@ideal-lab
# 调用：/baoyu-image-gen:baoyu-image-gen
```

**ideal-dify-generator** (v1.0.0) — Dify 工作流 DSL YAML 端到端生成与校验。从需求描述出发，自动生成符合 Dify DSL 规范的 YAML 配置文件，并做语法校验和逻辑检查。适用于需要快速搭建 Dify 工作流的场景。

```bash
claude plugin install ideal-dify-generator@ideal-lab
# 调用：/ideal-dify-generator:ideal-dify-generator
```

### 4.3 两大开发辅助

**ideal-ralph** (v0.6.0) — 苏格拉底式任务澄清 + 持久小步迭代验证循环。核心理念是"先想清楚再动手"。Ralph 通过对话引导你明确任务的输入、输出、验收标准和实施方式，生成任务合约（contract），然后以持久迭代循环逐步执行，每一步都验证是否满足验收标准。它不隶属于 ideal-dev-workflow，而是独立编排器，可以调用任何 Skill 完成工作。

```bash
claude plugin install ideal-ralph@ideal-lab
# 调用：/ideal-ralph:ideal-ralph
```

**ideal-lab-maintainer** (v1.1.0) — ideal-lab 仓库维护工具。提供四个核心能力：`commit`（自动检测改动、生成 changeset、bump 版本）、`add-skill`（新增 Skill 模板）、`release`（发布流程）、`validate`（校验所有插件结构）。面向 ideal-lab 的开发者和管理员。

```bash
claude plugin install ideal-lab-maintainer@ideal-lab
# 调用：/ideal-lab-maintainer:ideal-lab-maintainer
```

### 4.4 如何选择适合你的插件？

| 角色 | 推荐插件 | 起步建议 |
|------|---------|---------|
| **开发工程师** | ideal-dev-workflow + ideal-ralph + ideal-lab-maintainer | 先用 ralph 澄清一个开发任务的验收标准，再体验 dev-workflow 的完整流程 |
| **产品经理** | ideal-ppt-suite + ideal-deep-research | 先用 deep-research 做一次竞品调研，再用 ppt-suite 生成汇报 PPT |
| **技术写作** | ideal-document-workflow + ideal-knowledge-base | 先用 document-workflow 写一份简短文档，体验评审机制 |
| **项目经理** | ideal-deep-research + ideal-ppt-suite + ideal-document-workflow | 用 deep-research 做项目前期调研，用 ppt-suite 做里程碑汇报 |

## 5. 15 分钟上手指南

这一章带你从零开始，15 分钟内完成"安装 → 调用 → 获得交付物"的完整体验。

### 5.1 前置条件

在开始之前，确认你已经准备好以下环境：

- **Claude Code 已安装**：终端中运行 `claude --version` 可以输出版本号
- **Anthropic API Key 已配置**：Claude Code 首次启动时会引导你配置 API Key

如果以上条件已满足，直接进入下一步。

### 5.2 第一步：添加 Marketplace

Marketplace 是 ideal-lab 插件的分发源。添加一次，后续所有插件都可以从这个源安装：

```bash
claude plugin marketplace add https://github.com/MTleen/ideal-lab
```

预期输出（示意）：

```
✓ Marketplace "ideal-lab" added successfully
  Source: https://github.com/MTleen/ideal-lab
  Plugins available: 9
```

### 5.3 第二步：安装插件

根据你的需要选择安装。推荐从 document-workflow 或 deep-research 开始——它们上手门槛最低、效果最直观：

```bash
# 推荐：文档写作工作流（8 个 Skill）
claude plugin install ideal-document-workflow@ideal-lab

# 或者：深度调研工具
claude plugin install ideal-deep-research@ideal-lab

# 或者：PPT 全链路（9 个 Skill）
claude plugin install ideal-ppt-suite@ideal-lab
```

安装完成后，Skill 自动加载，无需额外配置。

### 5.4 第三步：执行你的第一个任务

以 document-workflow 为例，完整体验一次"从一句话到交付物"的过程。

**启动任务：**

```
/ideal-document-workflow:ideal-document-workflow 帮我写一份"团队代码评审规范"介绍文档，目标读者是事业部开发团队，篇幅 1500 字左右
```

**阶段 1 — 需求确认：** 工作流会先分析你的需求，向你确认几个关键点：

```
📋 需求分析完成，确认以下信息：

- 文档主题：团队代码评审规范
- 目标读者：事业部开发团队
- 篇幅要求：约 1500 字
- 写作风格：实用导向

确认无误？(y/n，或补充修改意见)
```

回复 `y` 进入下一阶段。

**阶段 2 — 大纲审核：** 工作流生成大纲后暂停，让你审核：

```
📋 大纲已生成：

1. 为什么要做代码评审
2. 评审流程（PR 提交 → 评审 → 修改 → 合并）
3. 评审检查清单
4. 常见问题与最佳实践

确认大纲？(y/n，或补充修改意见)
```

回复 `y` 或提出修改意见。大纲确认后，工作流自动进入写作阶段。

**阶段 3 — 等待写作：** 工作流并行写作各章节，期间不需要你操作。

**阶段 4 — 获得交付物：** 写作完成后，工作流输出最终文档，通常渲染为 Markdown 文件保存在你指定的路径。

整个过程约 5-10 分钟，你只需要在需求确认和大纲审核两个节点做决策。

### 5.5 进阶用法

**YOLO 模式自动推进：** 如果你信任 AI 的判断，不想逐阶段手动确认，可以在启动任务时附加指令：

```
/ideal-document-workflow:ideal-document-workflow YOLO 模式，帮我写一份...
```

YOLO 模式下工作流自动推进，直到完成或触发熔断（发现 P0 级问题才停下来等你处理）。

**多插件组合：** ideal-lab 的 Skill 可以自由组合。一个典型的工作组合是先调研再输出：

```
# 第一步：深度调研
/ideal-deep-research:deep-research 调研 2025 年主流 LLM 应用开发框架对比，standard 模式

# 第二步：基于调研结果生成 PPT
/ideal-ppt-suite:ideal-ppt-workflow 基于上面的调研结果，生成一份"LLM 应用开发框架选型"汇报 PPT
```

### 5.6 常见问题

**Q: 安装失败怎么办？**
确认你的 Claude Code 版本支持 Plugin 功能。运行 `claude --version` 检查版本。如果版本过旧，更新 Claude Code 后重试。另外确认网络可以访问 GitHub。

**Q: 如何更新已安装的插件？**
重新运行安装命令即可覆盖更新：
```bash
claude plugin install ideal-document-workflow@ideal-lab
```

**Q: 可以只安装单个 Skill 吗？**
目前不支持。插件是安装的最小单位，每个插件包含若干 Skill。但安装后你可以只使用其中某个 Skill，不会影响其他功能。

**Q: 输出格式支持哪些？**
文档类插件默认输出 Markdown。PPT 套件输出可编辑的 PPTX 文件。调研工具输出 Markdown 格式的调研报告。所有文本输出都可以根据需要指定其他格式。

**Q: 数据安全如何保障？**
所有任务在本地 Claude Code 环境中执行，代码和文档不会上传到 ideal-lab 仓库。插件本身只包含 Skill 定义（Markdown 文件）和脚本，不包含任何数据收集逻辑。API 调用走 Anthropic 官方通道。

## 6. 下一步

### 6.1 推荐起步路径

不同角色的同事可以从不同的路径起步。以下是各角色的 3 步建议：

**开发工程师：**
1. 安装 `ideal-ralph`，用它澄清一个你手头任务的验收标准——体验"先想清楚再动手"
2. 安装 `ideal-dev-workflow`，选一个小功能任务走一遍完整流程
3. 体验 `ideal-yolo` 的多视角自动评审，对比人工评审的差异

**产品/管理：**
1. 安装 `ideal-deep-research`，选一个你关心的行业话题做一次 standard 模式调研
2. 安装 `ideal-ppt-suite`，基于调研结果生成一份汇报 PPT
3. 尝试 YOLO 模式，体验从调研到 PPT 的全自动流程

**技术写作：**
1. 安装 `ideal-document-workflow`，选一个你正在写的文档走一遍完整流程
2. 重点关注评审环节——读者、作者、专家三个视角分别给出了什么反馈
3. 安装 `ideal-knowledge-base`，尝试从现有项目材料中构建知识库

### 6.2 反馈与共建

ideal-lab 是持续迭代的活系统。使用过程中如果你发现问题或有改进建议，有以下渠道：

- **问题反馈**：通过 GitHub Issue 描述你遇到的问题
- **Skill 共建**：如果你在日常工作中沉淀出了一套好用的流程，可以按照 ideal-lab 的 Plugin 结构封装为新的 Skill，通过 PR 提交
- **经验分享**：使用 ideal-lab 产出的文档、PPT、调研报告，本身就是最好的经验沉淀

最后一句话：ideal-lab 的每一个 Skill 都是从真实项目中长出来的，不是拍脑袋设计的。你在使用中发现更好的做法，就是下一次迭代的起点。
