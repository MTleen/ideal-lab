# 金字塔原理在 PPT 大纲中的应用

## 核心原则

### 1. 结论先行
每个部分以核心观点开篇。幻灯片标题不是标签，而是该页的核心信息。

**错误**：`市场分析`
**正确**：`中国市场年增长 23%，成为最大增量来源`

### 2. 以上统下
上层观点是下层内容的总结。每一页幻灯片标题 = 该页所有卡片内容的共同结论。

### 3. 归类分组
同一层级的内容属于同一逻辑范畴。同一页的卡片应围绕同一个核心观点展开。

### 4. 逻辑递进
内容按某种逻辑顺序展开：
- **时间顺序**：过去 → 现在 → 未来
- **结构顺序**：整体 → 部分 → 细节
- **重要性顺序**：最重要 → 次重要 → 补充
- **演绎顺序**：大前提 → 小前提 → 结论

## 在大纲中的具体应用

### 大纲层级映射

```
演示文稿整体（核心论点）
  ├── 第一部分（支撑论点 1）
  │     ├── 页面 1（次级论点 1.1）
  │     ├── 页面 2（次级论点 1.2）
  │     └── 页面 3（次级论点 1.3）
  ├── 第二部分（支撑论点 2）
  │     ├── 页面 4（次级论点 2.1）
  │     └── 页面 5（次级论点 2.2）
  └── 第三部分（支撑论点 3）
        ├── 页面 6（次级论点 3.1）
        └── 页面 7（次级论点 3.2）
```

### MECE 原则

各部分之间应满足：
- **M**utually **E**xclusive（相互独立）：没有重叠
- **C**ollectively **E**xhaustive（完全穷尽）：没有遗漏

## SCQA 框架

适用于开场和策略类演示：

| 要素 | 英文 | 含义 | 幻灯片 |
|------|------|------|--------|
| **S** | Situation | 大家都认可的背景 | 1-2 页背景铺垫 |
| **C** | Complication | 打破现状的冲突/挑战 | 1 页问题定义 |
| **Q** | Question | 自然引出的核心问题 | 过渡页 |
| **A** | Answer | 我们的解决方案 | N 页主体内容 |

## 叙事模式选择

| 模式 | 结构 | 适用场景 |
|------|------|---------|
| 问题→方案 | 痛点 → 根因 → 方案 → 收益 | 销售、提案 |
| SCR | 情境 → 冲突 → 解决 | 战略报告、管理汇报 |
| What/Why/How | 是什么 → 为什么 → 怎么做 | 培训、教程、产品介绍 |
| 时间线 | 过去 → 现在 → 未来 | 进展报告、发展历程 |
| 对比论证 | 观点 → 证据 → 反驳 → 结论 | 辩论、说服性演示 |

## 大纲生成检查清单

- [ ] 每页标题是一个完整的陈述（而非标签）
- [ ] 每页内容都支撑该页标题
- [ ] 页面之间的逻辑关系清晰
- [ ] 整体结构遵循金字塔的 MECE 原则
- [ ] 开场有吸引力（Hook）
- [ ] 结尾有明确的行动号召（CTA）
- [ ] 信息密度与 design-spec 的 density 设置匹配
