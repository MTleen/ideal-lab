# Content Rules — 内容与风格规则

本文件定义 PPT 内容创作的核心规则和风格规范。所有内容生成必须遵循这些规则。

---

## 1. 内容规则 (Content Rules)

### 规则 1: 尊重读者注意力 (Respect Reader Attention)

**原则**：读者的注意力是有限的稀缺资源，每页内容必须精炼。

- 每页只传达一个核心观点
- 删除不直接支撑核心观点的内容
- 使用视觉层次引导注意力，而非堆砌信息
- 避免让读者在页面中寻找关键信息
- 正文段落不超过 3 行

**反例**：
```
本季度我们在多个维度取得了进展。销售额增长了15%，
客户满意度提升到92%，新用户获取成本降低了8%，
团队规模扩大了20人，产品迭代了3个版本...
```

**正例**：
```
Q3 核心指标全面向好

📈 营收增长 15%
😊 客户满意度 92%
💰 获客成本 -8%
```

### 规则 2: 数据可追溯 (Data Traceability with Source Attribution)

**原则**：所有数据必须标注来源，确保可信度和可验证性。

- 每个数据点标注来源（机构名称 + 报告名称 + 时间）
- 非公开数据需说明数据获取方式
- 预测数据标注预测模型和置信区间
- 图表标题或注释中包含数据来源
- 来源标注格式：`数据来源：[机构名]《[报告名]》[年份]`

**示例**：
```
中国云计算市场规模达 6,800 亿元
数据来源：中国信通院《云计算白皮书》2025
```

### 规则 3: 自包含提示词 (Self-contained Prompts)

**原则**：每页内容必须独立完整，读者无需上下文即可理解。

- 每页标题独立表达本页核心信息
- 图表标题完整说明图表内容
- 缩写首次出现时提供全称
- 引用的概念给出简要解释
- 不依赖"上一页提到"来理解内容

### 规则 4: 无占位符 (No Placeholders)

**原则**：生成的所有内容必须是实际、有意义的内容。

- 禁止使用 "Lorem ipsum" 等填充文本
- 禁止使用 "[待补充]" "[XX]" "TODO" 等占位标记
- 禁止使用无意义的示例数据
- 如果真实数据不可用，使用合理的假设值并标注"示意数据"
- 图表中的数据必须是有意义的演示数据

## 2. 风格规则 (Style Rules)

### 规则 1: 叙事性标题 (Narrative Headlines, Not Labels)

**原则**：标题应传达核心信息，而非仅标注内容类型。

**反例**（标签式）：
```
市场规模
用户画像
竞品分析
财务数据
```

**正例**（叙事式）：
```
中国 SaaS 市场突破千亿规模
25-35 岁用户贡献 68% 营收
三大竞品合计市占率超 75%
ARR 连续 4 个季度保持 40%+ 增长
```

判断标准：读者仅看标题即可获得关键信息，无需阅读正文。

### 规则 2: 避免 AI 套话 (Avoid AI Cliches)

**原则**：内容应有人类专业写作者的语气和质感，避免机械化的 AI 表达。

**禁止使用的表达**：

| 类别 | 禁止用语 | 替代方案 |
|------|----------|----------|
| 总结过渡 | "综上所述"、"总而言之"、"总的来说" | 直接给出结论 |
| 强调 | "值得注意的是"、"需要指出的是" | 删除，直接呈现信息 |
| 重要性 | "至关重要"、"不可或缺"、"举足轻重" | 用数据或事实说明 |
| 通用修饰 | "显著的"、"大幅度的"、"明显的" | 给出具体数字 |
| 展望 | "展望未来"、"未来可期" | 给出具体预测或计划 |
| 连接 | "与此同时"、"不仅如此" | 使用更自然的过渡 |
| 夸张 | "革命性"、"颠覆性"、"史无前例" | 客观描述程度 |

**好的表达特征**：
- 具体而非抽象（"增长 42%" > "大幅增长"）
- 客观而非主观（"NPS 评分 72" > "客户非常满意"）
- 简洁而非冗余（"Q3 营收 2.3 亿" > "在第三季度我们的营业收入达到了2.3亿元"）
- 有观点而非中立（"建议优先投放 A 渠道" > "A 渠道和 B 渠道各有优劣"）

### 规则 3: 有意义的封底 (Meaningful Back Cover)

**原则**：封底不是"谢谢观看"，而是最后的行动号召。

**封底内容要求**：
1. 一个清晰的下一步行动 (Call to Action)
2. 或核心结论的最终重申
3. 或联系方式/二维码（如适用）

**反例**：
```
谢谢观看！
Thank you!
```

**正例 — 决策场景**：
```
下一步行动
□ 确定合作模式 → 本周五前
□ 签署框架协议 → 下周三前
□ 启动试点项目 → Q2 启动
```

**正例 — 汇报场景**：
```
核心结论
SaaS 业务已验证 PMF，建议加速投入
→ Q2 目标：ARR 突破 5,000 万
```

### 规则 4: 统一的视觉语言 (Consistent Visual Language)

**原则**：整个 PPT 的视觉元素必须风格一致。

- **图标**：统一风格（全 outline 或全 filled），不混用
- **图表**：统一配色和样式（坐标轴、网格线、标签位置）
- **色块**：统一圆角、阴影风格
- **图片**：统一处理方式（全有滤镜或全无滤镜）
- **线条**：统一粗细和颜色
- **形状**：统一使用矩形/圆角矩形或圆形，不混用

## 3. 格式规范 (Format Specifications)

### 3.1 默认规格

| 参数 | 默认值 |
|------|--------|
| 画幅比例 | 16:9 (1920×1080) |
| 页脚 | 无 |
| Logo | 无 |
| 页码 | 无 |

### 3.2 禁止项

以下元素在默认情况下禁止出现在 PPT 中：

| 禁止项 | 原因 |
|--------|------|
| 页脚 (Footer) | 占用空间、干扰视觉 |
| 公司 Logo | 除非用户明确要求 |
| 页码 | 增加视觉噪音（如需要可在演讲者备注中标注） |
| 日期/时间水印 | 不专业 |
| "机密"/"内部"水印 | 除非用户明确要求 |
| 装饰性分割线 | 简洁优先 |
| 过度动画 | 干扰信息传达 |

### 3.3 标题格式规范

- 封面标题：居中或左对齐，Hero 字号
- 章节过渡页标题：居中，H1 字号
- 内容页标题：左上角对齐，H1 字号
- 标题不加句号
- 标题不加标点符号（除非表达需要问号或感叹号）
- 标题不换行（超出时精简措辞）

### 3.4 正文格式规范

- 正文左对齐（不使用两端对齐）
- 列表使用项目符号或编号
- 每个列表项不超过 2 行
- 列表项之间间距 ≥ 行高 × 0.5
- 段落首行不缩进
- 中英文之间自动添加空格

### 3.5 数据展示规范

- 大数字使用千位分隔符（1,000,000）
- 百分比保留一致的小数位（42% 或 42.3%，不混用）
- 货币标注单位（¥ 1,200 万 / $ 500K）
- 时间范围标注完整（2024 Q1 - 2025 Q4）
- 图表坐标轴标注单位
- 关键数据使用 Accent 色高亮
