## 6. WCAG 2.1 AA 对比度要求

### 最低对比度标准

| 文字类型 | 正常文字 (AA) | 大号文字 (AA) | 增强标准 (AAA) |
|----------|---------------|---------------|----------------|
| 最小对比度 | 4.5:1 | 3:1 | 7:1 |
| 大号文字定义 | < 18pt / < 14pt bold | ≥ 18pt / ≥ 14pt bold | — |

### 常用组合对比度验证

| 前景色 | 背景色 | 对比度 | WCAG AA 正常文字 | WCAG AA 大号文字 |
|--------|--------|--------|------------------|------------------|
| `#212121` | `#FFFFFF` | 16.1:1 | PASS | PASS |
| `#757575` | `#FFFFFF` | 4.6:1 | PASS | PASS |
| `#9E9E9E` | `#FFFFFF` | 2.8:1 | FAIL | FAIL |
| `#FFFFFF` | `#005587` | 8.3:1 | PASS | PASS |
| `#FFFFFF` | `#2196F3` | 3.2:1 | FAIL | PASS |
| `#FFFFFF` | `#0076A8` | 5.3:1 | PASS | PASS |
| `#FFFFFF` | `#003F6C` | 11.2:1 | PASS | PASS |
| `#FFFFFF` | `#1976D2` | 4.3:1 | FAIL | PASS |
| `#212121` | `#F5F7FA` | 14.5:1 | PASS | PASS |
| `#FFFFFF` | `#D04A02` | 3.9:1 | FAIL | PASS |
| `#212121` | `#FFE600` | 14.7:1 | PASS | PASS |
| `#212121` | `#FF9800` | 2.7:1 | FAIL | FAIL |

### 使用规则

1. **正文文字**必须满足 WCAG AA 正常文字标准（≥ 4.5:1）
2. **大号标题**（≥ 18pt）必须满足 WCAG AA 大号文字标准（≥ 3:1）
3. 弱化文字 `#9E9E9E` 不可用于正文，仅用于装饰性文字
4. 橙色系 `#FF9800` 上的白色文字不符合 AA，需使用深色文字 `#212121`
5. 黄色背景 `#FFE600` 必须使用深色文字，不可使用白色

## 7. 配色方案生成规则

### 60-30-10 规则详解

```
60% Primary 主色 — 背景、大面积色块、页面基调
  → 选择低饱和度、高明度的颜色
  → 通常是品牌色的浅色变体或中性色

30% Secondary 辅助色 — 标题、图形元素、卡片边框
  → 选择中等饱和度的颜色
  → 通常是品牌色本身

10% Accent 强调色 — 按钮、重点标注、数据高亮
  → 选择高饱和度、对比强烈的颜色
  → 通常是互补色或品牌色的对比色
```

### 渐变生成规则

- 主渐变使用 135° 角（左上到右下）
- 渐变色差不超过同一色系的 2 个色阶
- 暗色风格渐变使用 180° 角（上到下），从深到浅
- 避免使用超过 3 个色标的渐变

### 配色验证清单

- [ ] Primary 与 Secondary 对比度 ≥ 3:1
- [ ] Secondary 上的白色/深色文字 ≥ 4.5:1
- [ ] Accent 不与 Secondary 冲突
- [ ] 中性色层级清晰（3 级以上）
- [ ] 功能色（正/警/负）语义明确
- [ ] 整体配色不超过 5 种主要颜色
