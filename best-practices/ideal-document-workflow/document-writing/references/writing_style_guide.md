# 写作风格规范

## 写作三避免

| 避免类型 | 错误示例 | 正确写法 |
|---------|---------|---------|
| **第一人称** | "我们选择微服务架构" | "微服务架构被选定" |
| **问句标题** | "为什么需要数据治理？" | "数据治理需求分析" |
| **负面表述** | "没有数据质量就没有..." | "数据质量是...的基础" |

## 语言三要求

| 要求 | 说明 | 示例 |
|------|------|------|
| **正式** | 避免口语化、比喻化、情感化 | 用"实现"而非"搞定" |
| **准确** | 数值带范围和来源，术语首次定义 | "准确率达到88%-92%（基于测试集验证，n=1200）" |
| **客观** | 避免主观判断和夸大表述 | 用数据说话，避免"非常好" |

## 证据三要素

| 要素 | 要求 | 示例 |
|------|------|------|
| **有对照** | 说明与什么对比 | "相比传统方案，延迟降低60%" |
| **有来源** | 数据来自何处 | "根据XX客户实际测试数据" |
| **可核验** | 提供验证方法 | "详见XX测试报告，测试条件：..." |

## 格式五规范

| 规范 | 要求 |
|------|------|
| **标题层级清** | 使用 X.Y.Z 格式，避免跳跃 |
| **图表编号明** | 表X-Y、图X-Y 格式 |
| **分隔不滥用** | 分隔线仅用于章节间 |
| **列表不过度** | 无序列表 ≤ 全文行数5%，优先段落叙述 |
| **表格不代文** | 文字论述为主，表格仅辅助总结 |

## 段落规范

| 要求 | 说明 |
|------|------|
| **段落长度** | 每段 3-5 句话为宜，避免单句成段 |
| **段落首句** | 通常为该段主题句，概括本段核心内容 |
| **段落连贯** | 句子之间逻辑连贯，使用衔接词（因此、然而、此外等） |

**错误示例：**
```
❌ 单句成段：
微服务架构是一种分布式架构风格。
它将应用程序拆分为多个独立的服务。
每个服务可以独立部署和扩展。

❌ 列表替代段落：
数据治理平台的核心功能包括：
- 数据采集
- 数据清洗
- 数据质量管理
```

**正确写法：**
```
✅ 段落叙述：
微服务架构是一种分布式架构风格，它将应用程序拆分为多个独立的服务，每个服务可以独立部署和扩展。这种架构风格的优势在于提高了系统的可维护性和可扩展性，同时降低了服务之间的耦合度。
```

## 常见问题修复

### 问题1：第一人称过多

```
❌ 原文：我们选择微服务架构，因为...
✅ 改为：微服务架构被选定，其优势在于...
```

### 问题2：问句标题

```
❌ 原文：##### 为什么需要这些数据？
✅ 改为：#### X.Y.Z 数据需求分析
```

### 问题3：口语化表述

```
❌ 原文：靠卫星补位
✅ 改为：通过遥感技术补充监测覆盖
```

### 问题4：量化证据不足

```
❌ 原文：准确率约90%
✅ 改为：准确率达到88%-92%（基于测试集验证，n=1200）
```

### 问题5：无序列表过度使用（重要）

```
❌ 原文：
Web端技术架构：
- 框架：Vue 3.x + TypeScript
- UI：Element Plus
- 状态：Pinia
- 构建：Vite 5.x

✅ 改为：
Web端采用Vue 3.x框架结合TypeScript语言开发，UI组件库选用Element Plus或Ant Design Vue。状态管理通过Pinia实现，构建工具采用Vite 5.x以提升开发效率。
```

### 问题6：以表代文（重要）

```
❌ 原文：
经评估：
| 维度 | 方案A | 方案B | 选择 |
|-----|-------|-------|-----|
| 性能 | 高 | 中 | A |
| 成本 | 高 | 低 | B |

✅ 改为：
本系统技术选型需综合考虑性能、成本、可维护性等因素。

**方案A分析**：该方案采用高性能架构，响应时间可达50ms以内，吞吐量提升40%。代价是硬件成本较高，约需增加30%预算。

**方案B分析**：该方案采用经济型架构，成本可控，部署简单。但响应时间约为200ms，吞吐量受限。

**综合评估与选择**：基于核心业务对实时性的严格要求，选择方案A作为主系统架构。

（表格作为辅助总结）
**表X-X 方案对比总结**
| 维度 | 方案A | 方案B | 选择 |
|-----|-------|-------|-----|
| 性能 | 高 | 中 | A |
| 成本 | 高 | 低 | - |
```

## 质量检查清单

### 风格检查
- [ ] 无第一人称（"我们"→方案主语）
- [ ] 无问句标题
- [ ] 无口语化表述
- [ ] 无负面表述

### 内容检查
- [ ] 内容与大纲一致
- [ ] 写作要点已覆盖
- [ ] 来源已标注
- [ ] 无事实错误
- [ ] 术语一致

### 量化证据检查
- [ ] 数值有范围
- [ ] 数值有来源
- [ ] 有对比参照

### 格式检查
- [ ] 无序列表 ≤ 全文5%
- [ ] 表格不替代文字论述
- [ ] 标题层级清晰（X.Y.Z）
- [ ] 段落长度适中（3-5句为主）
- [ ] 避免单句成段
