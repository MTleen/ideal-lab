# Markdown 转 Word 规范

## 1. 概述

本文档规定技术标书从 Markdown 转换为 Word 格式的标准和要求，确保转换后的文档符合正式投标文件的排版规范。

## 2. Word 文档格式标准

### 2.1 字体规范

| 元素 | 中文字体 | 英文字体 | 字号 | 颜色 |
|-----|---------|---------|------|------|
| 正文 | 宋体 | Times New Roman | 小四（12pt） | 黑色 |
| 一级标题 | 黑体 | Arial | 三号（16pt） | 黑色 |
| 二级标题 | 黑体 | Arial | 四号（14pt） | 黑色 |
| 三级标题 | 黑体 | Arial | 小四（12pt） | 黑色 |
| 表格内容 | 宋体 | Times New Roman | 五号（10.5pt） | 黑色 |

**重要**：
- 所有文字颜色必须统一为黑色（RGB: 0, 0, 0）
- 标题不得使用斜体
- 标题不得加粗过度（保持黑体默认粗细）

### 2.2 段落格式

| 元素 | 行距 | 首行缩进 | 段前 | 段后 |
|-----|------|---------|------|------|
| 正文 | 1.5 倍 | 2 字符 | 0 | 0 |
| 一级标题 | 1.5 倍 | 无 | 0 | 0 |
| 二级标题 | 1.5 倍 | 无 | 0 | 0 |
| 三级标题 | 1.5 倍 | 无 | 0 | 0 |
| 表格内容 | 单倍 | 无 | 0 | 0 |

### 2.3 表格格式

| 要求 | 说明 |
|-----|------|
| 表格对齐 | 居中 |
| 单元格对齐 | 居中 |
| 表头 | 加粗，居中 |
| 表格标题 | 表格上方，格式：**表X-Y 表格名称** |

### 2.4 页面设置

| 项目 | 值 |
|-----|-----|
| 纸张大小 | A4 |
| 页边距 | 上下左右各 2.54cm（1英寸） |
| 页眉页脚 | 根据需要设置 |

## 3. 脚本使用方式

### 3.1 脚本位置与依赖

转换脚本位于 `technical-proposal-merge` skill 的 `scripts/markdown_to_word.sh`。

**依赖**：
- `pandoc`：转换核心，`brew install pandoc`
- `python-docx`：格式后处理，`pip3 install python-docx`

### 3.2 输入模式

脚本支持两种输入模式：

**单文件模式**（直接转换单个 Markdown 文件）：
```bash
# 基本用法
./scripts/markdown_to_word.sh 合并内容.md

# 指定输出文件名
./scripts/markdown_to_word.sh -o 投标文件.docx 合并内容.md
```

**目录模式**（合并目录下所有 Markdown 文件后转换）：
```bash
# 合并 sections/ 目录下所有 .md 文件
./scripts/markdown_to_word.sh sections/

# 指定文件匹配模式和输出
./scripts/markdown_to_word.sh -f "*.md" -o 投标文件.docx 章节目录/
```

### 3.3 脚本处理流程

| 步骤 | 操作 | 说明 |
|------|------|------|
| 步骤1 | 合并/读取文件 | 单文件直接读取；目录模式按文件名排序合并 |
| 步骤2 | 预处理 Markdown | 清理 \\newpage、水平线、空行、表格格式 |
| 步骤3 | Pandoc 转换 | --from=markdown+pipe_tables+tex_math_dollars |
| 步骤4 | Word 后处理 | 字体、字号、行距、首行缩进、表格居中、表头加粗 |
| 步骤5 | 清理临时文件 | 删除中间产物 |

## 4. Markdown 源文件要求

### 4.1 表格格式

Markdown 表格必须使用标准的 pipe table 格式：

```markdown
**表1-1 表格标题**

| 列1 | 列2 | 列3 |
|-----|-----|-----|
| 内容1 | 内容2 | 内容3 |
| 内容4 | 内容5 | 内容6 |
```

**关键要求**：
- 表头行与分隔行之间**不能有空行**
- 分隔行格式：`|---|---|---|`（每列至少3个减号）
- 表格前后应有空行

### 4.2 标题格式

```markdown
# 第一章 标题
## 1.1 节标题
### 1.1.1 小节标题
#### 1.1.1.1 段落标题
```

**关键要求**：
- 一级标题使用中文数字格式：`第X章`
- 子标题使用阿拉伯数字格式：`X.Y`、`X.Y.Z`
- 标题前后应有空行

### 4.3 公式格式

**行内公式**：`$公式$`

**独立公式**：
```markdown
$$
公式内容
$$
```

**关键要求**：
- 公式前后应有空行
- 使用 LaTeX 语法

### 4.4 禁止的元素

以下元素在转换前必须清理：

| 元素 | 处理方式 |
|-----|---------|
| `\newpage` | 删除，改用一级标题自动分页 |
| `---`（水平线） | 删除 |
| 目录标记 | 删除，Word 中手动生成 |
| HTML 标签 | 转换为 Markdown 等效格式 |

## 5. 转换流程

### 5.1 标准转换步骤

```bash
./scripts/markdown_to_word.sh -o 输出文件名.docx 输入.md
```

脚本内部自动执行以下步骤：

1. **预处理 Markdown** — 清理 \\newpage、水平分隔线、空行规范化、表格格式修复
2. **Pandoc 转换** — `pandoc input.md -o output.docx --from=markdown+pipe_tables+tex_math_dollars --to=docx`
3. **Word 后处理** — python-docx 设置字体（宋体正文/黑体标题）、字号、行距、首行缩进、表格居中、表头加粗、一级标题前自动分页
4. **清理临时文件** — 删除中间产物

### 5.2 转换检查清单

| 检查项 | 要求 |
|-------|------|
| 字体 | 正文宋体，标题黑体 |
| 字号 | 正文小四（12pt），标题三/四号 |
| 颜色 | 全部黑色 |
| 行距 | 1.5倍 |
| 首行缩进 | 2字符 |
| 段前段后 | 0 |
| 标题斜体 | 无 |
| 表格 | 居中对齐，表头加粗 |
| 分割线 | 无 |
| 目录 | 无（手动添加） |

## 6. 常见问题与解决方案

### 6.1 表格未正确转换

**问题**：Markdown 表格在 Word 中显示为纯文本

**原因**：表格头行与分隔行之间有空行

**解决**：预处理正则自动修复
```python
content = re.sub(r'(\|[^\n]+\|)\n\n(\|[-:| ]+\|)', r'\1\n\2', content)
```

### 6.2 字体颜色不一致

**问题**：部分文字颜色不是黑色

**解决**：后处理统一设置 `run.font.color.rgb = RGBColor(0, 0, 0)`

### 6.3 标题显示为斜体

**问题**：Pandoc 转换后标题为斜体

**解决**：后处理中取消斜体 `run.italic = False`

### 6.4 出现 `\newpage` 文本

**问题**：Word 中显示 `\newpage` 字符串

**解决**：预处理时删除 `content = re.sub(r'\\newpage', '', content)`

## 7. 多章节合并

### 7.1 合并顺序

按照章节编号顺序合并，目录模式下脚本自动按文件名排序。

### 7.2 章节分隔

- 各章节之间使用空行分隔
- 不使用水平分隔线
- 一级标题自动触发分页

### 7.3 编号一致性

合并前检查：
- 章节编号连续（无跳跃）
- 子标题编号与章节号对应
- 表格编号与章节号对应（表X-Y）
