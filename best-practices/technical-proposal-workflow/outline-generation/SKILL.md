# Outline Generation (P3)

## Overview

根据需求分析结果，生成文档大纲，包含1-3级标题结构和篇幅估算。

**核心原则：按需调研，不强制但可用。**

## 输入

- P1-需求分析.md
- 项目目录和文档名称
- 原始材料路径（可选，用于本地调研）

## 调研决策

在生成大纲之前，根据需求判断是否需要调研：

### 判断逻辑

```
需要线上调研？
├── 竞品分析、技术趋势、行业背景 → 调用 ideal-deep-research
└── 不需要

需要本地调研？
├── 用户提供了源码/原始材料 → 读取并分析
└── 不需要（纯概念文档）
```

### 调研类型

| 类型 | 触发条件 | 执行方式 |
|------|---------|---------|
| **线上调研** | 竞品分析、技术选型、行业背景 | 调用 `ideal-deep-research` |
| **本地调研** | 用户提供了源码/原始材料路径 | 读取并分析源码/文档 |

## 调研执行

### 线上调研（调用 ideal-deep-research）

```bash
# 确定调研主题和范围
研究主题：{根据 P1 中识别的调研需求}
调研深度：standard

# 调用 ideal-deep-research
```

**调用时机：**
- P1-需求分析.md 中识别出需要外部信息的部分
- 技术选型需要对比分析
- 行业背景需要了解

### 本地调研（读取源码/原始材料）

```bash
# 读取用户提供的原始材料
源码路径：{raw_material_path}
文档路径：{文档目录}

# 分析内容包括：
# - 项目结构
# - 核心模块
# - 技术栈
# - 接口定义
# - 配置文件
```

**调用时机：**
- 用户提供了项目源码路径
- 用户提供了原始文档（设计文档、API文档等）

## 大纲模板

### 标准文档结构

```
1. 概述
   1.1 背景
   1.2 目标
   1.3 范围

2. 现状分析
   2.1 业务现状
   2.2 技术现状
   2.3 存在问题

3. 总体设计
   3.1 设计原则
   3.2 架构设计
   3.3 技术路线

4. 功能设计
   4.1 功能模块一
   4.2 功能模块二

5. 技术实现
   5.1 技术选型
   5.2 核心算法
   5.3 接口设计

6. 实施计划
   6.1 实施路径
   6.2 里程碑
   6.3 资源计划

7. 保障措施
   7.1 质量保障
   7.2 风险管控
   7.3 运维保障
```

**注：** 具体大纲结构应根据文档类型调整，上述为通用模板。

## 篇幅估算规则

| 章节层级 | 基础篇幅 | 复杂度加成 |
|---------|---------|------------|
| 一级标题 | 500-800字 | +200字/每个二级子节 |
| 二级标题 | 300-500字 | +150字/每个三级子节 |

## 输出格式

```markdown
# P2 大纲

## 文档信息
- 文档名称：{名称}
- 大纲版本：V1.0

## 调研摘要（如有）
### 线上调研
{ideal-deep-research 返回的关键发现}

### 本地调研
{从源码/原始材料中提取的关键信息}

## 大纲结构

### 1. 概述 (约800字)
#### 1.1 背景
[一句话概述]

#### 1.2 目标
[一句话概述]

...

## 篇幅汇总

| 章节 | 预估字数 |
|------|---------|
| 1. 概述 | 800 |
| 2. 现状分析 | 1200 |
| 3. 总体设计 | 1500 |
| ... | ... |
| **总计** | **10000** |

## 配图需求预估

| 位置 | 配图类型 |
|------|---------|
| 3.2 架构设计 | 系统架构图 |
| 4.1 功能模块 | 功能流程图 |
```

## 执行步骤

### Step 1: 读取 P1-需求分析.md

解析需求文档，识别：
- 需要调研的部分（竞品、技术选型、行业背景）
- 是否提供了原始材料路径

### Step 2: 判断调研需求

```
如果 P1 中有"需要线上调研"标记：
  → 调用 ideal-deep-research

如果用户提供了 raw_material_path：
  → 读取并分析源码/原始材料

如果都不需要：
  → 直接进入 Step 3
```

### Step 3: 生成大纲

基于需求分析和调研结果，生成文档大纲。

### Step 4: 估算篇幅

根据大纲结构，估算各章节篇幅。

### Step 5: 保存文件

保存到 `{项目目录}/文档产出/P2-大纲.md`
