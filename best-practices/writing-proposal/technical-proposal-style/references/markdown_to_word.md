# Markdown 转 Word 规范

## 1. 概述

本文档规定技术标书从 Markdown 转换为 Word 格式的标准和要求，确保转换后的文档符合正式投标文件的排版规范。

## 2. Word 文档格式标准

### 2.1 字体规范

| 元素 | 中文字体 | 英文字体 | 字号 | 颜色 |
|-----|---------|---------|------|------|
| 正文 | 宋体 | Times New Roman | 小四（12pt） | 黑色 |
| 一级标题 | 黑体 | Arial | 三号（16pt） | 黑色 |
| 二级标题 | 黑体 | Arial | 四号（14pt） | 黑色 |
| 三级标题 | 黑体 | Arial | 小四（12pt） | 黑色 |
| 表格内容 | 宋体 | Times New Roman | 五号（10.5pt） | 黑色 |

**重要**：
- 所有文字颜色必须统一为黑色（RGB: 0, 0, 0）
- 标题不得使用斜体
- 标题不得加粗过度（保持黑体默认粗细）

### 2.2 段落格式

| 元素 | 行距 | 首行缩进 | 段前 | 段后 |
|-----|------|---------|------|------|
| 正文 | 1.5 倍 | 2 字符 | 0 | 0 |
| 一级标题 | 1.5 倍 | 无 | 0 | 0 |
| 二级标题 | 1.5 倍 | 无 | 0 | 0 |
| 三级标题 | 1.5 倍 | 无 | 0 | 0 |
| 表格内容 | 单倍 | 无 | 0 | 0 |

### 2.3 表格格式

| 要求 | 说明 |
|-----|------|
| 表格对齐 | 居中 |
| 单元格对齐 | 居中 |
| 表头 | 加粗，居中 |
| 表格标题 | 表格上方，格式：**表X-Y 表格名称** |

### 2.4 页面设置

| 项目 | 值 |
|-----|-----|
| 纸张大小 | A4 |
| 页边距 | 上下左右各 2.54cm（1英寸） |
| 页眉页脚 | 根据需要设置 |

## 3. Markdown 源文件要求

### 3.1 表格格式

Markdown 表格必须使用标准的 pipe table 格式：

```markdown
**表1-1 表格标题**

| 列1 | 列2 | 列3 |
|-----|-----|-----|
| 内容1 | 内容2 | 内容3 |
| 内容4 | 内容5 | 内容6 |
```

**关键要求**：
- 表头行与分隔行之间**不能有空行**
- 分隔行格式：`|---|---|---|`（每列至少3个减号）
- 表格前后应有空行

### 3.2 标题格式

```markdown
# 第一章 标题
## 1.1 节标题
### 1.1.1 小节标题
#### 1.1.1.1 段落标题
```

**关键要求**：
- 一级标题使用中文数字格式：`第X章`
- 子标题使用阿拉伯数字格式：`X.Y`、`X.Y.Z`
- 标题前后应有空行

### 3.3 公式格式

**行内公式**：`$公式$`

**独立公式**：
```markdown
$$
公式内容
$$
```

**关键要求**：
- 公式前后应有空行
- 使用 LaTeX 语法

### 3.4 禁止的元素

以下元素在转换前必须清理：

| 元素 | 处理方式 |
|-----|---------|
| `\newpage` | 删除，改用一级标题自动分页 |
| `---`（水平线） | 删除 |
| 目录标记 | 删除，Word 中手动生成 |
| HTML 标签 | 转换为 Markdown 等效格式 |

## 4. 转换流程

### 4.1 标准转换步骤

1. **预处理 Markdown**
   - 清理 `\newpage` 指令
   - 清理水平分隔线
   - 确保表格格式正确
   - 确保标题格式正确

2. **使用 Pandoc 转换**
   ```bash
   pandoc input.md -o output.docx \
     --from=markdown+pipe_tables+tex_math_dollars \
     --to=docx \
     --mathml \
     --wrap=none \
     --standalone
   ```

3. **后处理 Word 文档**
   - 设置字体（宋体/黑体）
   - 设置字号
   - 设置行距（1.5倍）
   - 设置首行缩进
   - 设置表格居中
   - 清除段前段后间距
   - 清除标题斜体

### 4.2 转换检查清单

| 检查项 | 要求 |
|-------|------|
| 字体 | 正文宋体，标题黑体 |
| 字号 | 正文小四，标题三/四号 |
| 颜色 | 全部黑色 |
| 行距 | 1.5倍 |
| 首行缩进 | 2字符 |
| 段前段后 | 0 |
| 标题斜体 | 无 |
| 表格 | 居中对齐 |
| 分割线 | 无 |
| 目录 | 无（手动添加） |

## 5. 常见问题与解决方案

### 5.1 表格未正确转换

**问题**：Markdown 表格在 Word 中显示为纯文本

**原因**：表格头行与分隔行之间有空行

**解决**：
```python
# 预处理时修复表格格式
content = re.sub(r'(\|[^\n]+\|)\n\n(\|[-:| ]+\|)', r'\1\n\2', content)
```

### 5.2 标题编号格式错误

**问题**：章节编号不一致（如"第1章"与"第一章"混用）

**解决**：统一使用中文数字格式
```python
# 统一章节编号
content = re.sub(r'# 第\d+章', '# 第一章', content)
```

### 5.3 字体颜色不一致

**问题**：部分文字颜色不是黑色

**解决**：在后处理中统一设置
```python
run.font.color.rgb = RGBColor(0, 0, 0)
```

### 5.4 标题显示为斜体

**问题**：Pandoc 转换后标题为斜体

**解决**：后处理中取消斜体
```python
run.italic = False
```

### 5.5 出现 `\newpage` 文本

**问题**：Word 中显示 `\newpage` 字符串

**解决**：预处理时删除
```python
content = re.sub(r'\\newpage', '', content)
```

## 6. 多章节合并

### 6.1 合并顺序

按照章节编号顺序合并：
1. 第一章
2. 第二章
3. ...
4. 第十三章

### 6.2 章节分隔

- 使用空行分隔各章节
- 不使用水平分隔线
- 一级标题自动触发分页

### 6.3 编号一致性

合并前检查：
- 章节编号连续（无跳跃）
- 子标题编号与章节号对应
- 表格编号与章节号对应（表X-Y）

## 7. 脚本参考

通用转换脚本位于 skill 的 `scripts/` 目录，主要功能：

1. **合并章节**：按顺序合并多个 Markdown 文件
2. **格式清理**：预处理 Markdown 格式问题
3. **Pandoc 转换**：使用优化参数转换
4. **Word 后处理**：设置字体、行距、缩进等格式

使用方法：
```bash
./scripts/markdown_to_word.sh --output 技术方案.docx 章节目录/
```
