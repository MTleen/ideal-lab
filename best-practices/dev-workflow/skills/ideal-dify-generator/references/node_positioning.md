# 节点坐标布局规范

## 布局常量（来自 Dify 前端）

| 常量 | 值 | 说明 |
|------|----|------|
| NODE_WIDTH | 240px | 节点宽度 |
| X_OFFSET | 60px | 节点间距（留白） |
| NODE_WIDTH_X_OFFSET | 300px | 节点宽度 + 间距 |
| Y_OFFSET | 39px | Y 轴基准偏移 |
| START_INITIAL_POSITION | { x: 80, y: 282 } | Start 节点起始位置 |
| NODE_LAYOUT_HORIZONTAL_PADDING | 60px | 水平留白 |
| NODE_LAYOUT_VERTICAL_PADDING | 60px | 垂直留白 |
| NODE_LAYOUT_MIN_DISTANCE | 100px | 节点最小间距 |

## 水平布局（线性流程）

```
Start(80, 282) → LLM(380, 282) → Code(680, 282) → Answer(980, 282)
```

每步 X 坐标 = 前一步 X + 300，Y 坐标保持不变。

## 分支布局

### If-Else 分支

```
                    → True分支(680, 132)
IfElse(380, 300) ─────────────────────────→ Aggregator(980, 300)
                    → False分支(680, 468)
```

- True 分支 Y = 300 - 168 = 132
- False 分支 Y = 300 + 168 = 468
- Aggregator Y = 300（中间对齐）

### 多元分类（Question Classifier）

```
                    → 分类A(680, 80)
                    → 分类B(680, 240)
Classifier(380, 280) ──────────────────→ Aggregator(980, 280)
                    → 分类C(680, 400)
                    → 其他(680, 520)
```

### 嵌套分支

```
Start(80, 282) → IfElse1(380, 282)
                      → IfElse2(680, 132) → ...
                      → Handler(680, 432) → ...
```

## 错误处理布局

```
                    → 成功分支(680, 132) → Aggregator(980, 300)
Code(380, 300) ──────────────────────────────────────────────────────→ Answer(1280, 300)
                    → 失败分支(680, 468) → 恢复处理(980, 468) → Aggregator(980, 300)
```

## 迭代/循环布局

容器节点（Iteration/Loop）有特殊布局：

```
Iteration(380, 282)
  ┌─────────────────────────────┐
  │  内部: 24, 68 起算           │
  │  内宽: ~600px               │
  │  内高: ~300px               │
  └─────────────────────────────┘
  后续节点: x=980, y=282
```

## 布局检查清单

- [ ] Start 节点位于最左侧（x ≈ 80）
- [ ] 节点之间水平间距 ≥ 300px
- [ ] 分支节点 Y 坐标合理（±150-200px）
- [ ] 同级节点 Y 坐标相同
- [ ] 多分支汇合点（Aggregator/Answer）居中对齐
- [ ] 无节点坐标重叠
- [ ] 无节点超出画布左侧边界
- [ ] Iteration/Loop 容器足够容纳内部节点
